## 1. 文件用途

本文件用于描述项目从当前核心战斗 MVP 到第一章公开 Demo，再到正式版方向预留的完整开发路线。

它回答：

- 机制、内容、UI、技术、QA 和发布准备如何排序。
- `PRD-codex.md` 中原始 16 条推荐开发顺序分别落在哪些任务和阶段里。
- 哪些内容必须等前置阶段完成后再做。

它不替代：

- `AGENTS.md`：稳定协作规则、术语红线、范围红线。
- `docs/prd/`：产品事实源、玩法定义、内容规模和验收标准。
- `docs/tasks/PROJECT_PLAN.md`：当前任务看板、最近进度和下一步入口。
- `docs/prd/PRD-changelog.md`：需求范围和技术方向变更记录。

维护规则：

- 本文件记录中长期路线和阶段闸门，避免频繁记录流水。
- 当前进度、任务完成状态和下一步优先级仍更新到 `PROJECT_PLAN.md`。
- 若本文件与 PRD 冲突，以用户最新明确指令和 `docs/prd/` 为准。
- 若调整阶段范围、内容规模、核心机制或技术栈，必须同步更新 `PRD-changelog.md`。

## 2. 总体目标

第一阶段不是直接做完整游戏，而是按三层 MVP 推进：

| 层级 | 名称 | 目标 |
|---|---|---|
| 阶段 A | 核心战斗 MVP | 一场战斗可玩、可测试、可复盘，跑通破形、问名、正名、伏诛 / 归册 |
| 阶段 B | 教学纵切 MVP | 连续 3 到 5 场战斗，验证机制解锁、奖励、朱批和最小局内成长 |
| 阶段 C | 第一章公开 Demo | 完整第一章试玩，含路线、商店、休整、事件、精英、Boss、法宝、结算 |

正式版方向暂按 4 个常规章 + 1 个终审短章预留，但当前开发和公开 Demo 只承诺第一章。

## 3. 核心开发原则

- 始终先保证核心规则可运行、可测试、可回放，再扩内容量和表现。
- 每次只推进一个明确模块，不一次性实现完整游戏。
- `src/core` 保持纯逻辑，不依赖 React、Phaser、DOM、浏览器存储或渲染对象。
- 卡牌、敌人、遭遇、法宝、事件、裁定尽量数据驱动。
- 新增卡牌、敌人或遭遇时，不应要求修改 React 组件或 Phaser Scene。
- 所有关键状态变化必须进入 `actionLog`。
- 伏诛路线能通关，归册路线奖励更高；正名不是第二套胜利条件。
- 封势只能处理来势，不能处理异动，这个边界必须在代码层成立。
- 法宝是牌组外器物，法宝本体不进入抽牌堆。

## 8. 阶段 C：第一章公开 Demo

阶段目标：完成对外试玩级第一章，单局约 30 到 45 分钟。

内容目标：

| 模块 | 公开 Demo 目标 |
|---|---:|
| 可玩角色 | 1，执簿者 |
| 卡牌 | 42 到 48 |
| 普通敌人单位 | 12 |
| 低成本杂祟 / 召唤物 | 4 |
| 普通遭遇模板 | 18 到 22 |
| 精英敌人 | 3 |
| 事件精英 | 可选 1 |
| Boss | 1，窃榜使 |
| Boss 行为变体 | 2 到 3 |
| 法宝 | 10 到 12 |
| 事件 | 8 到 10 |
| 路线、商店、休整、结算、设置、存档 | 完整可用 |

推荐拆分：

### C1 内容池扩展

- 卡牌从 6 张扩到 15 到 20 张，再逐步扩到 42 到 48 张。
- 普通敌人先扩到 3 到 5 个，再扩到 12 个。
- 普通遭遇模板先做 10 到 12 个，再扩到 18 到 22 个。
- 每次新增内容必须有唯一 ID，并尽量通过数据文件接入。

### C2 路线与节点

- 建立简易路线图，再迭代成榜面节点地图。
- 节点类型包括普通战、精英、事件、商店、休整、Boss。
- 第 4 场之后允许在已解锁机制范围内使用随机遭遇模板。

### C3 成长系统

- 完整奖励页。
- 裁定三类完整 MVP：登簿、朱批、削籍。
- 墨、劫数、榜裂逐步加入。
- 削籍收益必须诱人，但榜裂代价必须可感知。

### C4 三坛

- 先做人坛，再做地坛，最后做天坛。
- 人坛：回合结束触发，适合把本回合问名、正名、朱批结果转为追加效果。
- 地坛：敌人行动前触发，适合封势、断供、净卷、禁术、阻止遮名。
- 天坛：下回合开始触发，适合提前布置问名、抽牌、墨、敕令、归册条件。
- 三坛不能做成普通延迟破形或延迟封势。

### C5 法宝

- 法宝作为牌组外器物显示在法宝栏或榜册边缘。
- 先做未认主和认主，再做反噬。
- 反噬使用“连续两场战斗”的判定窗口，不是反噬持续两场。
- 法宝本体不进入抽牌堆；法宝可以生成临时牌，但临时牌不等于法宝本体。

### C6 精英与 Boss

- 精英保持 3 个，优先做扎实。
- Boss 只做 1 个“窃榜使”，通过 2 到 3 个行为变体体现路线差异。
- Boss 不做饶恕助战，不做敌魂卡，不做神将收集。

阶段 C 完成闸门：

- 第一章单局约 30 到 45 分钟。
- 至少 3 条构筑方向能闭合。
- 玩家能清楚解释问名和归册为什么有收益。
- 路线、商店、休整、事件、精英、Boss、设置、存档、继续游戏、结算页可用。
- 不包含授权不清的素材。
- `npm run typecheck`、`npm run test`、`npm run build` 形成发布前验证基线。
