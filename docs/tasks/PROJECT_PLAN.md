## 3. 进度更新规则

每完成一个编号任务或阶段性开发任务，都必须更新本文件。

更新要求：

- 已完成的任务状态改为“已完成”。
- 当前推荐下一步保持唯一且明确。
- 若新增任务，写入目标、范围、不做内容和验收标准。
- 若本次任务改变验证基线，更新“当前验证基线”。
- 若任务范围、核心机制、技术栈或阶段规划发生变化，同时更新 `docs/prd/PRD-changelog.md`。
- 若只是代码实现或测试补齐，没有改变需求范围，不需要更新 changelog。

推荐收尾顺序：

1. 完成代码或文档改动。
2. 运行该任务需要的验证命令。
3. 更新本文件的任务状态、当前下一步和验证基线。
4. 如有需求变化，再更新 `docs/prd/PRD-changelog.md`。
5. 最终回复中说明本文件已更新。

## 5. 阶段 A 收口记录

### T07：来势 / 封势 / 异动边界

状态：已完成
优先级：P0
目标：把敌人行动压力做实，并在代码层保证封势不能处理异动。

范围：

- 将来势从日志反馈推进为可被封势处理的战斗状态。
- 实现至少一个异动，例如偷香或塞污卷。
- 实现至少一张专门应对异动的卡牌或效果。
- UI 或调试面板明确区分来势与异动。
- 补测试：封势能处理来势，但不能处理异动。

不做：

- 不做完整敌人池。
- 不做路线、商店、事件。
- 不做完整三坛或法宝。

验收：

- 来势有数值。
- 封势只降低来势。
- 异动能执行。
- 封势不能影响异动。
- actionLog 记录来势、封势、异动执行或反制。

完成记录：

- `EnemyState.incomingForce` 成为当前来势状态，封势会扣减该值。
- 偷香鼠提供偷香异动，未反制时降低下一回合香火。
- 断供符提供专门断异动效果，能阻止偷香。
- UI 显示当前来势、异动预警，并支持在测试战斗中切换纸面鬼 / 偷香鼠。
- 新增 `src/tests/enemyIntent.test.ts` 覆盖来势、封势、异动执行和反制。

### T10：简易奖励与机制解锁

状态：已完成
优先级：P0  
目标：让连续战斗有最小成长反馈。

范围：

- 战后简易奖励页。
- 伏诛 / 归册奖励质量差异。
- 根据教学进度解锁新卡或机制。
- 未解锁机制不进入奖励池。

不做：

- 不做完整商店。
- 不做完整法宝奖励。
- 不做正式平衡。

完成记录：

- 新增 `src/core/run/rewardResolver.ts`，根据伏诛 / 归册生成普通或高质量奖励。
- 伏诛奖励提供 2 个普通卡牌选项，归册奖励提供 3 个选项，并允许出现归册限定奖励牌。
- 奖励池按 `unlockState` 过滤，未解锁的来势 / 异动、朱批入口相关牌不会提前进入奖励。
- 教学 run 状态新增跨战斗 `deckDefinitionIds`、`pendingReward` 和 `rewards`，选牌会加入后续战斗牌组，跳过会记录但不改牌组。
- HUD 新增战后奖励页，胜利后需选择或跳过奖励，再进入下一场教学战。
- 新增 `src/tests/rewardResolver.test.ts` 覆盖奖励质量差异、解锁过滤、选牌入牌组和跳过奖励。
