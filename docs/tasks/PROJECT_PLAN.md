若本文件与 PRD 或用户最新指令冲突，按 `AGENTS.md` 中的事实源优先级处理。

## 3. 进度更新规则

每完成一个编号任务或阶段性开发任务，都必须更新本文件。

更新要求：

- 已完成的任务状态改为“已完成”。
- 当前推荐下一步保持唯一且明确。
- 若新增任务，写入目标、范围、不做内容和验收标准。
- 若本次任务改变验证基线，更新“当前验证基线”。
- 若任务范围、核心机制、技术栈或阶段规划发生变化，同时更新 `docs/prd/PRD-changelog.md`。
- 若只是代码实现或测试补齐，没有改变需求范围，不需要更新 changelog。

推荐收尾顺序：

1. 完成代码或文档改动。
2. 运行该任务需要的验证命令。
3. 更新本文件的任务状态、当前下一步和验证基线。
4. 如有需求变化，再更新 `docs/prd/PRD-changelog.md`。
5. 最终回复中说明本文件已更新。

## 5. 阶段 A 剩余规划

### T07：来势 / 封势 / 异动边界

状态：待做  
优先级：P0  
目标：把敌人行动压力做实，并在代码层保证封势不能处理异动。

范围：

- 将来势从日志反馈推进为可被封势处理的战斗状态。
- 实现至少一个异动，例如偷香或塞污卷。
- 实现至少一张专门应对异动的卡牌或效果。
- UI 或调试面板明确区分来势与异动。
- 补测试：封势能处理来势，但不能处理异动。

不做：

- 不做完整敌人池。
- 不做路线、商店、事件。
- 不做完整三坛或法宝。

验收：

- 来势有数值。
- 封势只降低来势。
- 异动能执行。
- 封势不能影响异动。
- actionLog 记录来势、封势、异动执行或反制。

### T08：战斗 UI 补强

状态：待做  
优先级：P0  
目标：让玩家在 UI 上清楚理解来势、封势、异动的区别。

范围：

- 显示当前来势值。
- 显示异动预警。
- 显示封势处理结果。
- 显示异动是否被专门效果处理。
- 保持最小可用，不追求正式美术。

不做：

- 不做完整动画、音效、正式美术。
- 不做路线页、奖励页、裁定页。

### T09：前三场教学战纵切

状态：待做  
优先级：P0  
目标：连续跑通前 3 场固定教学战。

战斗规划：

- 第 1 场：纸面鬼，教学形、破形、问名、正名、伏诛、归册。
- 第 2 场：偷香鼠，教学来势、封势、异动、断供。
- 第 3 场：铜铃夜巡，教学朱批入口或朱批雏形。

范围：

- 固定战斗顺序。
- 简单战后推进。
- 每场只引入必要机制。
- 奖励和解锁可以先做最小版本。

不做：

- 不做完整路线地图。
- 不做随机遭遇池。
- 不做商店或事件链。

### T10：简易奖励与机制解锁

状态：待做  
优先级：P0  
目标：让连续战斗有最小成长反馈。

范围：

- 战后简易奖励页。
- 伏诛 / 归册奖励质量差异。
- 根据教学进度解锁新卡或机制。

不做：

- 不做完整商店。

- 不做正式平衡。
