# 第一章三角色 QA 与数值护栏 T63

日期：2026-05-21

## 目标

T63 用于确认 T62 新增的衡简、照微、莲烬不是只停留在开局入口，而是都能进入并完成第一章基础路线。本轮先建立机器可复核护栏和 QA 记录，避免三角色扩展破坏第一章可通关基线。

## 版本口径

- 每个角色固定 1 件初始法宝。
- 衡简：己形 72，初始法宝为打神鞭残节。
- 照微：己形 64，初始法宝为照骨镜。
- 莲烬：己形 78，初始法宝为赤绫火轮。
- 三角色版本开局不再出现 starter 法宝三选一。
- 第一章途中和 Boss 后仍各保留 1 次法宝三选一。
- 通关时预期总持有 3 件法宝：角色初始法宝 + 途中选择 + Boss 后选择。

## 本轮范围

- 覆盖三角色 pure-logic 完整路线 smoke。
- 覆盖待处理奖励、裁定、朱批、法宝 offer 的自动清理。
- 覆盖三角色起始法宝、法宝 offer 阶段和通关摘要。
- 不新增角色、不扩章节、不调整核心机制。
- 不以本轮测试替代人工试玩手感判断。

## 角色观察点

| 角色 | 本轮机器护栏 | 后续试玩重点 |
|---|---|---|
| 衡简 | 完整第一章可收束；通关持有 3 件法宝；后续 offer 为途中 / Boss 后 | 稳健基准是否过于平均，打神鞭残节是否足够体现“旧榜执笔”的裁断感 |
| 照微 | 完整第一章可收束；照骨镜固定起始；低己形不阻断机器路线 | 64 己形下首战和前两场是否压迫过高，问名看行动是否真的让玩家感觉有决策收益 |
| 莲烬 | 完整第一章可收束；赤绫火轮固定起始；后续 offer 节奏不变 | 连打爆发是否过强，躁火反噬是否可理解，直攻路线是否把问名 / 归册完全挤掉 |

## 验证记录

- `npm run test -- src/tests/chapterOneLongFlow.test.ts` 通过：1 个测试文件，4 个测试。
- `npm run verify` 通过：lint、typecheck、全量 Vitest、production build 均通过。
- 全量 Vitest 当前为 27 个测试文件、150 个测试。
- production build 仍出现 Vite chunk size warning；该风险沿用 T61 / T60A 记录，本轮不处理 code splitting。
- `git diff --check` 通过。

## 剩余风险

- pure-logic smoke 证明路线和 run 状态能收束，不证明每个战斗手牌体验已经好玩。
- 赤绫火轮的真实连打强度依赖战斗内出牌选择，本轮只把完整路线和法宝节奏纳入护栏。
- 照微低己形是否需要再补初始牌或提示，需要结合人工试玩反馈判断。
