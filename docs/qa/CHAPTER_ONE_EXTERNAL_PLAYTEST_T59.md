# T59 外部试玩反馈模板与小范围试玩准备

日期：2026-05-20
阶段：阶段 D 第一章完整可玩硬化
版本标识：`chapter-one-playable-t59-2026-05-20`

## 1. 本轮目标

本文件用于把第一章交给少量外部试玩者前，统一试玩说明、反馈格式、已知问题边界和传播限制。

本轮只做小范围试玩准备，不公开宣传，不做 Steam 页面，不做媒体素材，也不把试玩反馈视为正式版承诺。

## 2. 试玩目标

请试玩者重点回答三个问题：

1. 第一章是否能从“新开本局”完整玩到第一章结算，或至少能清楚知道自己为什么失败。
2. 玩家是否理解问名、正名、归册、裁定、三坛、法宝和反噬为什么值得关注。
3. 哪些页面、文案、数值、反馈或环境步骤最容易卡住。

建议小范围先找 3 到 5 名可信试玩者，覆盖至少：

- 1 名熟悉牌组构筑肉鸽的玩家。
- 1 名不熟本项目术语但愿意读提示的玩家。
- 1 名普通浏览器环境使用者，用于检查启动、窗口、音量和本地存档。

## 3. 试玩启动说明

### 3.1 开发者准备

在发起试玩前，开发者应先执行：

```bash
npm install
npm run build
npm run preview -- --host 127.0.0.1
```

打开终端显示的本地地址，通常是：

```text
http://127.0.0.1:4173/
```

如果是同机试玩或屏幕共享，由开发者打开该地址即可。如果把仓库或压缩包交给试玩者本机运行，试玩者也按以上命令启动。

### 3.2 试玩者操作

| 目标 | 操作 |
|---|---|
| 从头开始 | 标题页点击“新开本局”。这会清掉当前路线进度。 |
| 继续上次进度 | 标题页点击“继续游戏”。当前只恢复路线节点和跨战状态，不恢复战斗中途瞬时状态。 |
| 调整音量或动画 | 标题页点击“设置”。可静音、调音量、关闭动画。 |
| 重开当前战斗 | 战斗页点击“重开当前战斗”。 |
| 完全重置本地记录 | 优先点击“新开本局”；如需彻底清理，可清浏览器站点数据。 |

浏览器本地记录 key：

```text
bangwairen.tutorial.save.v2
bangwairen.tutorial.save.v1
bangwairen.settings.v1
bangwairen.playtest.errorLog.v1
```

## 4. 试玩任务单

试玩者不需要追求最优打法，请按自然理解推进：

1. 从“新开本局”开始。
2. 至少完成前三场教学战，记录是否理解问名、正名和归册收益。
3. 尽量推进到一次精英战和一次商店 / 休整节点。
4. 如果能完成第一章，记录 Boss 是伏诛还是归册，以及最后是否进入第一章结算。
5. 如果失败或中途放弃，记录卡在哪里、为什么停下。

建议记录：

```text
试玩者代号：
版本标识：
操作系统：
浏览器与版本：
屏幕分辨率：
是否使用鼠标：
是否使用静音：
是否关闭动画：
试玩时长：
推进结果：完成第一章 / 战斗失败 / 中途放弃 / 页面卡住
最终页面：
```

## 5. 问题分类

反馈问题至少归入以下一类：

| 分类 | 定义 |
|---|---|
| 阻断 bug | 无法继续、无法结算、页面崩溃、按钮无响应、存档坏掉、战斗无法结束。 |
| 理解问题 | 不知道下一步做什么，不理解术语，不理解为什么要问名 / 归册 / 裁定 / 法宝三选一。 |
| 数值问题 | 过难、过易、回合拖太长、奖励无吸引力、己形压力或榜裂压力不清楚。 |
| 表现问题 | 反馈不明显、声音刺耳或太弱、动画干扰、焦点不清楚、画面遮挡或溢出。 |
| 文案问题 | 文字不顺、术语混乱、错别字、提示和实际效果不一致。 |
| 发布 / 环境问题 | 启动失败、白屏、加载慢、浏览器兼容、音频无法播放、本地存档异常。 |

## 8. 已知问题与非承诺范围

这些不是“已修复”，试玩者遇到时仍可反馈影响程度：

- 战斗中刷新页面后，继续游戏会恢复到当前路线战斗的开场状态，不恢复该回合手牌、抽弃牌、敌人形等瞬时状态。
- 当前仍是浏览器试玩版本，不包含桌面安装包、Steam 页面、自动更新、云存档或多存档槽。
- 美术和音频仍是低成本占位，不代表正式视觉和正式音乐质量。
- `npm run build` 可能继续出现 Phaser 首包超过 500 kB 的 Vite warning；本阶段只在试玩反馈出现明显加载卡顿时再单独列优化任务。
- 当前反馈模板用于收集问题，不代表后续正式版功能承诺。

已修复内容不要混入已知问题清单。若试玩者反馈的问题已在当前版本中无法复现，应单独标为“待复核”，不要直接标“已修复”。

## 9. 传播限制

本轮小范围试玩不公开传播：

- 不发布公开截图、视频、直播、社交媒体内容或媒体素材。
- 不把试玩包转发给未约定的人。
- 不把当前占位美术、占位音频或截图当作正式宣传素材。
- 不使用“封神版杀戮尖塔”“类杀戮尖塔”“AI 生成游戏”等宣传表达。

如需对外公开任何素材，必须另开发布 / 宣传任务，复核素材授权、文案边界和版本说明。

## 10. 反馈整理口径

收集反馈后建议按以下表格整理：

| ID | 分类 | 严重度 | 页面 / 系统 | 现象 | 复现状态 | 当前处理 | 后续任务 |
|---|---|---|---|---|---|---|---|
| 待填 | 待填 | 待填 | 待填 | 待填 | 待填 | 待填 | 待填 |

整理时优先顺序：

1. P0 阻断 bug。
2. P1 理解问题和关键页面卡点。
3. 明显影响完整局的数值问题。
4. 表现、文案和环境问题。
5. 低优先级建议。
