## T86 追加调整

用户确认“这里面需要高压线补给与压力微调有哪些，还有后续修复的，直接在 T86 进行修改”。本轮将原后续 T87 候选项直接并入 T86，实际调整如下：

- 裂榜塞卷商店线：`route_node_fracture_shop_fouled_pressure` 不再继续抽 `塞卷吏再审` 与 `瘟纸人与塞卷吏`，改为污染、瘟纸人再审、供桌残像或供桌 + 偷香鼠压力，避免照微在已经历塞卷旧案和商店后再次被连续塞卷压穿。
- 归册事件商店线：`污牒收束` 后新增 `终审前校册` 休整节点，复用既有休整系统，给火部迟审 + 污牒收束之后、Boss 之前留下修形窗口。
- 窃榜使 Boss 开场：归册路线即使带 `high_pressure`，也优先走归册线的遮名 / 遮势开场；只有明确 `fracture` 或没有路线家族但带高压标签时，才走裂榜撕榜塞卷开场。
- 照微奖励倾向：照微的雷符、鞭击、破形与阵法权重小幅上调，让低己形洞察角色在高压线更容易拿到实际收束牌，而不是只堆问名 / 坛位。
- 重复 key 修复：牌组追加 ID 不再用当前牌组长度直接生成，改为读取已有最大编号后递增；读取旧存档时会修复重复 `run_card_*` ID；商店 / 休整 / 朱批牌册列表也用索引兜底，避免旧存档残留重复 key 时继续触发 React warning。

## 经济观察

- 钱不是当前高压线的主要瓶颈：照微失败样本结算仍有 56 / 32 香火钱，衡简失败样本仍有 85 香火钱。
- 商店取舍已成立：本轮出现了香封、删牌、朱批服务和买卡的真实购买选择，没有只剩无脑买卡。
- 稳行线偏宽松：莲烬买 3 次商店后仍以 92 香火钱、23 / 78 己形通关，说明低压路线经济余量较足。
- 高压线的问题更像“资源来得及买，但来不及把压力转换掉”：照微两次都在 Boss 前倒下，衡简能进 Boss 段但最终被压穿。

## 香封观察

- 香封能真实出现、携带和消耗：衡简线精英后候选出现镇势 / 护名、护名 / 回香，莲烬线出现断异 / 焚案；商店也能买到镇势或焚案。
- 2 槽边界目前可读：出现过槽满替换，例如衡简商店购买镇势香封时替换护名香封。
- 多敌战里香封有明确价值：衡简多敌战消耗护名和镇势，莲烬多敌战消耗焚案和断异，能缓解遮名、偷香或多目标压力。
- 问题在于高压线香封不一定能成为救命窗口：照微复核中仍持有 1 枚香封但倒在 5 / 9，说明裂榜塞卷线压力可能早于香封发挥窗口。

## 秘传观察

- 卡牌奖励候选里秘传偏稀：照微主样本 0，衡简 1，莲烬 2；照微复核仍为 0。
- 秘传没有压平 Boss：莲烬即便看到 2 个秘传候选，也只是 23 / 78 通关；衡简看到 1 个秘传候选仍失败。
- 当前更像“惊喜位偏保守”，不是“秘传过强”。后续若要调，优先看高压线是否需要更稳定的基础补强，而不是先削秘传。

## 多敌压力观察

- 新多敌模板真实进入中后段，不会挤进前三场固定教学。
- 衡简的逃名纸马 + 香灰虱能通过，但会明显消耗香封和己形窗口；遮名 + 小偷香组合读得懂，目标选择也清楚。
- 莲烬遇到灰坛童子 + 铜铃夜巡、偷香鼠 + 香灰虱两场多敌，群体破形牌和焚案 / 断异香封都有实际价值，最终仍能通关。
- 多敌模板本身不是立即压死人的问题；真正危险的是多敌、塞卷 / 污卷、精英和 Boss 连续叠在同一路线时，没有足够修形或缓冲窗口。

## 结论

已直接调入 T86：

- 照微裂榜塞卷商店线下限过窄：已收敛商店后污染节点遭遇池，避免连续塞卷；已上调照微破形收束倾向。
- 归册事件商店线 Boss 前后压力偏紧：已在事件商店高压线 Boss 前补休整；已修正归册 + 高压路线被误判成裂榜 Boss 开场的问题。
- 浏览器控制台重复 key：已修复牌组追加 ID、旧存档重复 ID 归一化和牌册列表 key 兜底。

可观察：

- 秘传出现频率偏低但可见。当前不建议先改秘传强度；若后续高压线仍窄，优先增加基础补强可见性。
- 香封 2 槽读得懂，且真实消耗成立。后续更值得看“香封进入时机”和“高压线可用窗口”，不是先改槽位上限。
