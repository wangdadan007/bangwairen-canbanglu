# T65 第一章路线 / 角色 / 法宝构筑倾向补强记录

日期：2026-05-22
适用范围：第一章三角色扩展阶段 E，承接 T64 三主线路线分支

## 1. 本轮目标

T65 不是新增内容量，而是把 T64 后仍偏“能跑通”的中间体验补成更像肉鸽爬塔的可规划选择：

- 路线选择要让玩家看懂“这条路会把本局推向什么方向”。
- 奖励要开始贴近衡简、照微、莲烬各自的长期构筑倾向。
- 途中和 Boss 后法宝三选一要成为构筑锚点，而不是固定列表。
- Boss 对路线倾向有轻量响应，但不做强惩罚、不做角色锁路。

## 2. 实现范围

- 路线页展示路线倾向标签与可达节点提示：稳行、归册、裂榜、补给、高压。
- 战后奖励排序加入角色倾向权重：
  - 衡简偏归册、朱批、抽牌、问名和墨。
  - 照微偏问名、断异动、来势识别、天坛和抽牌。
  - 莲烬偏破形、香火、劫数 / 榜裂、法宝联动。
- 法宝 offer 加入角色与路线倾向权重：
  - 保留三角色固定起始法宝。
  - 保留第一章途中和 Boss 后各一次三选一。
  - 候选排序会随角色和已走路线倾向变化。
- 窃榜使 Boss 开场来势加入轻量路线响应：
  - 归册线更可能先出遮名压力。
  - 裂榜 / 高压线更可能先出塞污卷压力。
  - 其他路线保持原先按册来势开场。
- 法宝 offer 页显示每件法宝的构筑锚点标签，帮助玩家识别偏问名归册、偏破形节奏、偏朱批、偏裁定或偏封势。

## 3. 不做内容

- 不新增第 4 角色。
- 不新增第二章、新 Boss 池或完整随机地图。
- 不改变形、问名、正名、伏诛 / 归册、裁定三类等核心机制。
- 不改变法宝不入牌堆规则。
- 不改变当前法宝节奏：角色固定起始法宝 + 途中三选一 + Boss 后三选一。
- 不把任何路线做成某个角色的专属路线。

## 4. 机器验证

本轮已新增或更新以下测试点：

- `rewardResolver.test.ts`：验证同一高质量奖励池会按衡简、照微、莲烬给出不同首选倾向，且选项数量不变。
- `artifactResolver.test.ts`：验证途中和 Boss 后法宝 offer 会根据角色与路线倾向调整首位候选。
- `eliteBossEncounter.test.ts`：验证窃榜使 Boss 能按路线倾向切换初始 intent。
- `chapterOneLongFlow.test.ts`：长流程 smoke 使用路线倾向上下文生成法宝 offer，确保三角色三主线仍能完整收束。

## 5. 当前结论

T65 达成的是“可规划性补强”，不是最终手感定稿。机器测试证明：

- 三角色仍能完成第一章，不会因倾向排序造成 pending choice 卡死。
- 路线倾向现在不仅被记录，还会影响法宝候选和 Boss 开场压力。
- 角色差异从起始牌组 / 起始法宝扩展到奖励排序和法宝构筑锚点。

## 6. 剩余风险

- 真实手牌节奏仍需人工试玩确认，尤其是照微低己形在裂榜线的 Boss 前容错。
- 法宝 offer 目前是确定性排序，不是正式随机池；后续若要做完整爬塔感，需要再设计 seeded offer 和稀有度。
- Boss 只做开场 intent 调整，不是完整 Boss 变体；若后续继续强化，必须保留补给抵消空间。
- 路线页提示是低成本文案与标签，后续可结合结算摘要记录玩家实际路线风格。

## 7. 下一步建议

T66 建议做三角色人工试玩复核：每个角色至少选一次非推荐主线，记录 Boss 前己形、Boss 后己形、奖励是否像角色方向、法宝是否能形成选择纠结，以及路线提示是否足够让玩家预判风险。
