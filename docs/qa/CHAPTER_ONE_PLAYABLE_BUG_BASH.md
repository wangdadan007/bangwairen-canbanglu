## 1. 目标

本轮用于确认第一章是否能通过浏览器内真实点击从新游戏推进到 Boss 与第一章结算，而不是只依赖纯逻辑 smoke。

重点检查：

- 伏诛完整局是否能通关。
- 偏归册完整局是否能通关。
- 削籍 / 榜裂路线是否能通关。
- 法宝反噬或临时反噬牌是否会阻断战斗。
- 失败 / 重开和继续游戏路径是否可用。
- 是否存在无法继续、无法结算、奖励 / 裁定 pending 卡住、坏档、页面崩溃或战斗无法结束。

## 3. 失败 / 重开 / 继续游戏

| 路径 | 结果 |
|---|---|
| 放弃本局 | 通过。点击 `放弃本局` 后出现第一章结算页，并提供 `重开第一章`。 |
| 重开第一章 | 通过。结算页点击 `重开第一章` 后回到第一战：纸面鬼。 |

非阻断说明：

- 当前存档保存 run / route，不保存战斗内临时状态。若在战斗中打出一张牌后刷新，继续游戏会回到同一路线战斗的开场状态，而不是恢复该回合的敌人形、手牌和香火。T51 未将其视为 P0 阻断；后续 T52 / T58 可决定是否在 UI 或试玩说明中明确此限制。
- 当前战斗结算后，伏诛 / 归册入口已经可继续推进，但下方仍保留敌方面板、手牌和 `结束回合` 按钮，卡牌会显示“当前没有可选目标”。这不阻断流程，但容易让玩家误以为还需要继续操作，建议在 T52 调整为更清晰的战后状态。
- 已补入己形最小闭环：己形 72 / 72 起步，敌方未封来势会扣己形，己形归零进入战斗失败；己形跨战斗保留，休整可修形。T51 当时的登簿己形上限收益已在 2026-05-24 平衡补丁中移除，当前普通登簿改为“残名入档”通用规则：本幕段最多 3 次，每次每场首次正名 +1 墨。

## 4. 本轮发现与处理

| 编号 | 类型 | 现象 | 处理 |
|---|---|---|---|
| T51-B01 | UI 稳定性 | 路线进度列表遇到重复 encounter id 时，浏览器控制台出现 React duplicate key error：`encounter_pool_offering_table_afterimage_return`。 | 已修复。`BattleHud` 的路线进度列表 key 改为 `encounterId + index`，不改变路线数据或玩法逻辑。 |
| T51-B02 | 路线一致性 | 第四战节点标题为“无灯庙祝”，但该节点带 encounter pool 时可能抽到“偷香鼠 + 香灰虱”等其他遭遇；旧存档的 `run.encounterIds` 也会让战斗进度列表继续显示旧遭遇。 | 已修复。第四战固定为 `encounter_mid_unlit_temple_warden`；resolver 会忽略固定节点上的旧存档抽样结果，载入 / 热更新时会用当前 route 战斗顺序同步 run 进度列表；中后段普通战节点继续保留 encounter pool 抽样。 |
| T51-B03 | 文案可读性 | 卡牌名为“叩天坛”，规则文本开头写“奉天坛”，玩家不易判断这是卡名还是效果类型。 | 已修复。规则文本改为“布置天坛：下回合开始时问名 1，并获得 1 墨”，保留三坛机制含义。 |
| T51-B04 | 战斗压力 | 玩家没有己形，敌方来势只产生日志和节奏压力，无法形成跨战损耗或失败条件。 | 已修复。己形 72 / 72 起步并跨战保留；未封来势扣己形，归零败退；休整新增修形。T51 当时的登簿己形上限收益已在 2026-05-24 平衡补丁中移除，当前普通登簿改为“残名入档”通用规则：本幕段最多 3 次，每次每场首次正名 +1 墨。 |
| T51-B05 | 卡池节奏 | 问名源在前中后段偏多，精英 / Boss 名格数量有限时容易溢出。 | 已处理。实际问名入口压到 8 张直接问名 + 1 张天坛间接问名；照骨札、天迹敕、镜契敕、随鞭符转为抽牌 / 墨 / 破形联动。 |

P0 阻断问题：

- 本轮未发现无法继续、无法结算、奖励 / 裁定 pending 卡住、坏档、页面崩溃或战斗无法结束。
