# T102 归册裁定分层压力统计

日期：2026-05-27；2026-05-28 同步断名符削籍口径

## 目标

验证 T102 后“普通怪只开放朱批 / 削籍，精英 / Boss 只开放登簿”的实际 run 压力：

- 普通怪归册不再生成通用登簿。
- 全普通归册 + 优先朱批不会无限滚出朱批次数。
- 全普通归册 + 优先削籍会产生可见榜裂 / 牌组 / 墨资源机会成本。
- 纯伏诛、混合归册、极端归册都能在 run 流程层收束 pending 选择，不把极端路线判定为必须最优。

## 方法

新增 `src/tests/t102VerdictPressure.test.ts`，走真实第一章 route / run resolver：

- 覆盖 3 个角色：衡简、照微、莲烬。
- 覆盖 6 条固定路线支线：稳行商店、稳行休整、归册双精英、归册事件、裂榜事件、裂榜商店。
- 覆盖 5 种裁定策略，共 90 条纯逻辑完整路线样本。
- 朱批 offer 不跳过，自动给当前可朱批牌写入第一个可见主朱批，用于观察实际朱批承载量。
- 奖励、事件、休整、途中 / Boss 后法宝 offer 都按现有长流程 smoke 口径自动清理。

边界：这不是回合级战斗 AI，也不是浏览器手动通关证明；它验证的是当前 run 层裁定、奖励、资源与路线压力是否出现明显数值膨胀或 pending 死锁。

| 纯伏诛 | 18 | 0 | 0 | 0 | 0 | 0 | 0-1 | 11-13 | 0 | 0 | 0-1 | 0 / 18 |
| 混合归册 | 18 | 4-5 | 1-2 | 0 | 3-4 | 0 | 3-4 | 11-13 | 0-1 | 0 | 0-1 | 18 / 18 |

| 全归册削籍：断名符 | 18 | 8-9 | 2-3 | 0 | 0 | 5-7 | 0-1 | 16-22 | 0-1 | 0 | 5-8 | 18 / 18 |

说明：

- “实际朱批写入”包含裁定朱批，也包含休整等既有朱批服务，因此可能高于“普通朱批裁定”。
- 纯伏诛样本中最多 1 次朱批写入来自非裁定服务，不是普通归册收益。
- 专属登簿只来自精英 / Boss；通用登簿全样本为 0。

## 结论

- T102 分层合同成立：90 个样本里普通 `register_common_docket` 生成次数为 0；精英 / Boss 登簿稳定保留。
- 全普通归册优先朱批的普通朱批裁定为 5-7 次，不是无限膨胀；但实际总朱批可到 8 次，仍是后续试玩要重点感受的高构筑路线。
- 断名符削籍路线沿用原“基础削籍爆发”压力区间，会把牌组推到 16-22 张，并积累 5-8 榜裂；这个机会成本比朱批路线清楚，不像无脑必选。
- 抄裂得墨削籍路线的墨达到 9-15，是当前最需要继续观察的资源峰值；它没有直接涨牌组，但能转化为后续朱批 / 休整服务，应在真实试玩里看是否过于舒服。
- 极端路线在 run 层都能完成，不代表它们是实战最优；本轮统计只说明“没有系统性失控或死锁”，是否压过正常混合路线仍需要本地试玩体感验证。

## 后续观察

- 本地试玩优先看全朱批路线：5-8 次实际朱批是否让核心牌过早成型。
- 本地试玩优先看断名符路线：它替代旧裂形符削籍后，“目标已有名被揭示”条件是否让削籍爆发更有辨识度。
- 本地试玩优先看抄裂得墨路线：9-15 墨是否让“先削籍再朱批 / 休整”过于顺滑。
- 若上述任一路线在真实战斗里明显压过混合路线，再单独调整普通朱批节奏限制、抄裂得墨数值或榜裂阈值反馈；不需要把登簿重新塞回普通怪归册。
