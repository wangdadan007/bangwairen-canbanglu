# T91/T92 真实流程截图复核与小范围试玩取样准备

日期：2026-05-25

## 1. 范围

本轮是 T91 首局理解 / Boss 前准备感与 T92 战斗表现纵切完成后的复核任务。

已完成：

- 用真实浏览器页面流程采集标题页、首战、路线分流、商店准备、休整准备和 Boss 开场截图。
- 覆盖 1280x720、1366x768、1920x1080 三档桌面视口。
- 基于截图复核修复一个 HUD 顺序问题：当前战斗 / 商店 / 休整面板不再被路线总览和法宝栏压到首屏下方。
- 整理小范围真人试玩取样表，供后续 3 到 5 人内测记录。

未完成：

- 本轮没有真人外部反馈样本，不把自动化脚本结果冒充为玩家试玩结论。
- 本轮不进入正式美术资产量产、复杂动画、正式音频或 Steam 宣传素材制作。

## 3. 截图复核

### 标题页

- 3 个视口均能看到标题、三角色入口、继续 / 设置 / 残榜预览等外层入口。
- 未发现横向溢出。

### 首战

- 低视口 1280x720 下，右侧 HUD 首屏已能看到首局提示、敌方目标、敌形、名格、来势 / 异动信息。
- 敌方剪影和 Phaser 桌案非空，但左侧战斗桌案仍显偏空，敌方视觉锚点集中在舞台下方。
- 手牌 / 操作区在 1280x720 下仍可能需要右侧 HUD 内部滚动，后续若做 T93，应优先处理战斗首屏密度和操作区可见性。

### 路线分流

- 前三战后路线分流可达，路线页不再出现在战斗中抢占首屏。
- 这是 T91 准备感摘要能被看见的前提。

### 商店准备

- 商店页首屏可见“补给整册”，包含己形、墨 / 香火钱、问名 / 防守、收束牌、香封、法宝 / 改榜等状态块。
- 商品、牌册和离开按钮可继续滚动查看；没有横向溢出。

### 休整准备

- 休整页首屏可见“休整整册”，能解释修形、删牌、朱批和法宝维护取舍。
- 对比商店页，休整页的信息密度更稳定，低视口下阅读压力较小。

### Boss 开场

- Boss 开场可见 Boss 标签、归册 / 异动提示、敌方剪影、敌形、名格和异动说明。
- 1280x720 下仍存在“先看到敌方状态，再向下滚动看手牌”的问题；这是表现 / 布局二期候选，不是规则阻断。

## 4. 已修复问题

问题：T91/T92 截图复核发现，`BattleHud` 在战斗、商店、休整等当前操作页之前总是先展示 run 总览、路线页和法宝栏，导致关键面板在低视口首屏里被下推。

修复：

- `BattleHud` 只在路线选择、事件占位、商店占位、休整占位等路线节点页展示完整 `RoutePage`。
- 战斗、奖励、裁定、朱批、事件、商店、休整等当前操作状态下，不再把 `TutorialRunPanel` 和顶部 `ArtifactBar` 放在关键面板之前。

效果：

- 商店 / 休整的 T91 准备摘要可以在首屏露出。
- 战斗和 Boss 开场优先展示敌方状态和当前行动压力。
- 核心规则、数值、卡牌、敌人、路线和经济均未改变。

## 5. 小范围试玩取样表

本表用于后续 3 到 5 人内测。当前状态是“准备完成，等待真人样本”，不计为已完成外部试玩。

| 样本 | 目标玩家 | 场景 | 重点观察 | 状态 |
|---|---|---|---|---|
| A | 熟悉卡牌肉鸽玩家 | 莲烬首战到商店线 Boss 前 | 是否能用补给整册判断删牌、香封、朱批和 Boss 风险 | 待采样 |
| B | 不熟悉项目术语玩家 | 衡简首战到休整线 Boss 前 | 是否能理解形、问名、正名、归册、临战整册 | 待采样 |
| C | 低分辨率 / 笔记本玩家 | 1280x720 首战、商店、Boss 开场 | 是否频繁找不到手牌、按钮或当前目标 | 待采样 |
| D | 偏视觉反馈玩家 | 中段战、精英或 Boss 开场 | 敌人剪影、破形 / 正名 / 异动反馈是否足够记忆点 | 待采样 |

建议提问：

- 第一战里，你最先理解的是“打形”还是“问名 / 正名”？
- 进商店或休整时，你是否知道 Boss 前最缺什么？
- 香封、朱批、删牌、修形这几项，哪一项最难判断价值？
- Boss 开场时，你是否能看出它接下来是来势还是异动压力？
- 低视口下，哪一块信息最容易被错过？

## 6. 结论

T91/T92 的基础方向成立：Boss 前准备摘要确实应该优先露出，T92 的程序化桌案和敌方剪影已经能提供非空视觉锚点。

但当前还不适合直接进入正式美术量产。下一步更稳的是先做小范围真人试玩取样；若反馈与截图复核一致，T93 应聚焦战斗首屏密度、敌方锚点位置、手牌 / 操作区可见性和少量关键反馈强化，而不是新增玩法系统。
