# T94 美术动画与游戏画面表现二期记录

日期：2026-05-25  
任务：T94 美术动画与游戏画面表现二期  
性质：离屏可推进的规格、占位表现、截图验收与资产清单，不进入正式美术量产

## 1. 本轮目标

T94 承接 T92 战斗表现垂直切片一期与 T92A 真实流程截图经验，把“画面好不好看”先拆成可远程执行、可复核的几件事：

- 明确第一章战斗画面二期的视觉层级。
- 强化敌方视觉锚点、当前行动和手牌 / 操作区的扫读关系。
- 给正式素材生产前的对象、规格、授权要求和优先级留台账。
- 新增一条可重复运行的截图验收脚本，覆盖三档常用视口。
- 只使用项目内 CSS / Phaser 程序化占位，不新增生产依赖、第三方素材或正式音频。

## 2. 覆盖页面与画面目标

| 页面 / 状态 | T94 目标 | 验收观察点 |
|---|---|---|
| 标题页 | 保留残榜主题首屏识别，不在本轮重做品牌资产 | 标题、角色选择和继续入口不溢出 |
| 首战战斗页 | 敌方锚点、当前行动、手牌 / 操作区同屏可扫 | 敌方区域、来势 / 异动、手牌和日志不互相遮挡 |
| 第二战战斗页 | 观察问名、封势、后一动等信息密度上升后的可读性 | 当前行动和后一动不被敌形、名格或反馈挤掉 |
| 中段路线 / 多敌前后 | 观察路线信息和战斗信息切换是否稳定 | 低视口下无横向滚动，按钮文字完整 |
| Boss 开场 | 强化终审压迫感，但不改变 Boss 机制 | Boss 锚点、来势 / 异动、路线压力提示可见 |
| 裁定页 | 登簿 / 朱批 / 削籍仍是三类判印式选择 | 削籍基础判词与法宝强化分组可扫读 |
| 结算页 | 保持本局改榜记录清楚，不在本轮做新终局演出 | 伏诛 / 归册 / 裁定留痕不混成普通战绩 |

## 3. 战斗画面层级

T94 的首屏顺序按“先判断危险，再决定出牌”整理：

1. 敌方锚点：敌形 / 精英 / Boss 的位置、选中目标和正名阶段必须先被看见。
2. 当前行动：来势、异动、遮势和后一动要比装饰更醒目。
3. 己方可行动区：香火、手牌、按钮不可用原因和当前目标要连在一起。
4. 长期状态：三坛、法宝、榜裂、劫数、墨和香封保持可查，但不抢当前回合焦点。
5. 记录层：actionLog 负责解释发生了什么；动画关闭时仍可靠日志和静态短提示读懂。

## 6. 截图验收清单

运行方式：

```bash
npm run dev -- --host 127.0.0.1
npm run capture:t94
```

输出：

- `docs/qa/screenshots/t94-presentation-phase-two/manifest.json`
- `1280x720`、`1366x768`、`1920x1080` 三档视口截图

每张截图重点看：

- `horizontalOverflow` 是否为 `false`。
- `canvas.present` 是否为 `true`，且 `width / height` 大于 0。
- 战斗页 `enemyVisualCuesVisible`、`intentBannerVisible`、`handListVisible` 是否符合该页面状态。
- 标题页、路线页和裁定页不要求手牌可见，但不应出现按钮文字截断或横向滚动。
- 若 `06-verdict-page` 未生成而出现 `06-verdict-probe-fallback`，说明本轮自动路线未稳定进入归册裁定，需要人工补一张裁定页截图或后续强化脚本。

## 8. 剩余风险

- 程序化案面仍是占位，不代表正式美术方向定稿。
- 自动脚本能验证布局和露出，不能替代用户审美判断或真人试玩反馈。
- 裁定页本轮已被脚本稳定捕捉；若后续卡牌或首战节奏调整导致未进入归册，脚本会记录 fallback，不伪装成真人复核。
- 低视口下信息密度仍高，后续若继续做 T94A，应重点看敌方面板和手牌区是否需要进一步折叠。

## 9. 模板沉淀判断

本轮产生了“表现规格 + 截图验收包 + 程序化占位增强”的通用流程，适合记录为后续 `React Card Game UI Kit` 与 `AI Game Development Planning and QA Kit` 候选；具体封神题材视觉、术语和裁定文案不进入通用模板。
