## 3. T58 浏览器试玩启动说明

首次准备：

```bash
npm install
npm run build
npm run preview -- --host 127.0.0.1
```

启动后打开终端显示的本地地址，通常是 `http://127.0.0.1:4173/`。

试玩操作：

| 需求 | 操作 |
|---|---|
| 从头开始 | 标题页点“新开本局”。这会清掉当前本地路线进度。 |
| 继续上次进度 | 标题页点“继续游戏”。当前存档恢复到路线节点，不保存战斗中途瞬时状态。 |
| 调整音量 / 静音 / 动画 | 标题页点“设置”。静音后战斗反馈不出声；关闭动画后 HUD 反馈不再播放动态强调。 |
| 普通重开当前战斗 | 战斗页点“重开当前战斗”。 |
| 完全清空本地记录 | 优先点“新开本局”；如需重置设置和错误日志，可清浏览器站点数据或删除下方 localStorage key。 |

需要清理的浏览器本地 key：

```text
bangwairen.tutorial.save.v2
bangwairen.tutorial.save.v1
bangwairen.settings.v1
bangwairen.playtest.errorLog.v1
```

## 4. 问题反馈格式

反馈问题时尽量带上：

```text
页面：例如标题页 / 战斗页 / 商店页 / 结算页
操作：刚才点了什么、选择了什么、是否刷新过页面
错误信息：如果页面显示“试玩错误记录”，直接贴出其中内容
期望：你觉得这里应该发生什么
实际：实际发生了什么
是否能复现：能 / 不能 / 不确定
```

前端错误记录方案：

- React 渲染错误会进入错误边界，页面会显示“试玩错误记录”和可复制的问题信息。
- 运行时错误与未处理的异步错误会写入 `bangwairen.playtest.errorLog.v1`，最多保留最近 8 条。
- 错误信息包含页面、时间、错误来源、错误文本、堆栈和浏览器信息；试玩者仍需要补一句“刚才做了什么”。

## 6. Phaser Chunk Warning 处理决定

`npm run build` 仍可能出现 Vite 的 Phaser 首包超过 500 kB 提示。

本阶段处理决定：记录为可接受风险，暂不做 code splitting。

原因：

- 当前目标是小范围本地浏览器试玩，不是公网大流量发布。
- Phaser 体积主要来自既有引擎依赖，本轮没有新增生产依赖。
- 现在拆分首包会额外增加路由、资源加载和预览部署变量，风险大于收益。
- 若 T59 外部试玩反馈出现首屏加载明显卡顿，再把 code splitting 或资源懒加载单独列为后续优化。
