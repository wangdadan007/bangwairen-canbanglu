# T95 真实游戏画面结构设计一期

日期：2026-05-26  
任务：T95 真实游戏画面结构设计一期  
性质：竞品战斗界面拆解、首屏结构定义与可执行布局方案；不做正式美术，不改运行 UI

## 1. 目标

T95 只解决一个问题：第一章战斗页到底应该像一张真实可玩的卡牌肉鸽战斗页，而不是继续保持“左侧演出台 + 右侧长 HUD”的开发面板结构。

本轮产出：

- 按销量 / 市场表现优先，拆 5 个同类或近邻头部作品的战斗 / 核心出牌界面。
- 定义本作战斗首屏结构：敌人、手牌、资源、来势 / 异动、三坛、法宝、日志各自放在哪里。
- 给 T96 提供可执行布局方向，并明确推荐采用“横向卡牌骨架 + 残榜审案中轴”的混合方案。

不做：

- 不新增玩法系统、卡牌、敌人、Boss、角色、章节或生产依赖。
- 不做正式敌人立绘、角色立绘、Steam 素材、正式字体或正式音频。
- 不引入授权不清素材；本轮只引用公开商店页 / 新闻页面做结构研究。
- 不把本作宣传写成“封神版杀戮尖塔”或“类杀戮尖塔”。

## 3. 竞品结构拆解

### 3.1 Slay the Spire

首屏结构：

- 敌人与玩家角色都在中上 / 中部舞台，敌方行动意图直接压在敌人附近。
- 手牌固定在底部，抽牌堆、弃牌堆、费用和结束回合按钮围绕手牌形成操作区。
- 遗物、金币、生命等长期状态压在顶部边缘，不占战斗中心。
- 没有常驻长日志；战斗结果通过实体附近的数值、图标、buff 和短动效反馈。

可学：

- 战斗首屏第一眼必须看到“谁要打我 / 我能打什么 / 我手上能出什么”。
- 资源和牌堆数字靠近手牌，不应该远离操作区。
- 长日志不是主界面的一列，它只服务解释和回看。

不可照搬：

- 本作敌人有“形 / 名 / 正名 / 现形反扑 / 来势 / 异动”多层信息，不能只用一个意图图标解决。
- 本作的仪式感来自残榜、朱批、三坛和法宝，不能做成纯西式怪物站台。

### 3.2 Slay the Spire 2

首屏结构延续初代，但画面更强调敌我体量、角色剪影和动效空间。它没有为了新内容把核心战斗界面拆成左右两张表，而是继续保持中心战场与底部手牌的基本骨架。

可学：

- 续作仍保留玩家已经理解的界面语法：敌方在上，手牌在下，状态在边缘。
- 新系统和更强表现不能破坏首屏判断链。
- 若要做更丰富的角色 / 敌人表现，应扩大中部战场，而不是把主要信息塞进右侧 HUD。

不可照搬：

- 本作不应直接追求《杀戮尖塔》式英雄对怪物站位；我们更像“案前审名”，玩家是执簿者，不是站在画面中部挥剑的主角。

### 3.3 Balatro

首屏结构：

- 牌桌是绝对中心，手牌和出牌区域一眼可见。
- Joker、消耗品、分数、盲注等状态都围绕牌桌分区，信息密度高但分层固定。
- 日志几乎不存在，结算通过数字爆发、牌面状态和短动效完成。

可学：

- 高销量卡牌游戏不一定需要大角色演出，但必须让“桌面操作”非常清楚。
- 周边状态可以多，但每一类都有固定槽位，不随着流程把页面挤高。
- 视觉主题可以强烈，但主题不应该妨碍玩家读牌和读资源。

不可照搬：

- Balatro 的核心是算分牌桌，本作有敌方行动压力，必须保留敌人 / 来势 / 异动的对峙层。

### 3.4 Inscryption

首屏结构：

- 玩家坐在桌前，对手在远端，牌局像一次被注视的仪式。
- 常驻 UI 很少，压力来自桌面物件、对手姿态、牌位和音画反馈。
- 回看信息弱，但即时状态靠桌面位置和物件隐喻承担。

可学：

- 本作国风基调不应该只靠皮肤颜色，而应该让玩家感觉自己坐在案前审名、翻榜、落印。
- 日志应降权，关键事件要落到桌面实体：名格显现、朱批落纸、来势压案、异动警示。
- 少量强仪式物件比一大列文字面板更能建立记忆点。

不可照搬：

- Inscryption 故意隐藏和压迫，本作第一章 Demo 必须保持卡牌肉鸽的公平可读，不能牺牲来势 / 异动预警。

### 3.5 Monster Train

首屏结构：

- 战场是多层垂直空间，敌我单位和路线直接承担玩法。
- 手牌仍在底部，资源 / 回合 / 结束按钮围绕出牌区。
- 多目标和多层压力在战场内部解决，而不是放进长 HUD 表格。

可学：

- 多敌、多状态并不等于右侧长面板；可以把目标选择和危险预告放回战场对象本身。
- 如果本作后续多敌更多，敌人应在同一案面上形成“压案队列”，每个敌人自带小型意图 / 名格 / 形条。

不可照搬：

- 本作目前不是车厢 / 多层塔防，三坛是时间轴槽位，不是三层战场。

## 4. 本作首屏结构原则

T96 需要先解决结构，不急着解决正式美术。首屏信息按以下优先级排列：

1. 当前危险：敌人、敌形、来势 / 异动 / 遮势、Boss 压案。
2. 当前操作：手牌、香火、目标、结束回合、不可用原因。
3. 本作核心差异：名格、正名状态、三坛、法宝。
4. 局内资源：己形、墨、劫数、榜裂、抽弃耗数量、香封。
5. 回看解释：日志、完整 tooltip、长说明。

明确结论：

- 敌人应该在画面上方中线或中上方，不应藏在右侧面板。
- 手牌必须横向铺在底部，不应是一列竖向按钮。
- 香火应靠近手牌；己形、墨、劫数、榜裂可以在底部操作区上沿或左下角，但不能远离当前决策。
- 来势 / 异动必须贴近敌人视觉锚点：来势用“可封的压案方向”，异动用“需断的朱红警示 / 印记”；遮势用虚线、雾层或半透明，不显示具体数值。
- 后一动应低于当前行动，作为小型预告，不抢主视觉。
- 三坛放在敌人和手牌之间的中线，表达战斗时间轴；人坛、地坛、天坛是三个稳定槽位。
- 法宝贴右侧或残榜边缘，像器物挂位；不要混入手牌，也不要占据主战场。
- 日志降权为折叠抽屉或最近两条短记录；完整日志可展开，但默认不占据首屏纵向空间。

## 5. 可执行布局方案

### 推荐方案：卡牌肉鸽横向骨架 + 残榜审案中轴（T96 执行稿）

T96 不采用纯方案 A，也不直接采用纯方案 B。

推荐执行口径是：用方案 A 的横向卡牌肉鸽骨架保证首屏可读性，用方案 B 的审案中轴确立本作调性。

原因：

- 纯方案 A 更稳，但容易变成常见“敌人在上、牌在下”的西式怪物站台，残榜、裁定、执簿者身份不够强。
- 纯方案 B 更像本作，但若第一版就铺满左右辅栏，容易重新变成“面板夹战场”的开发 HUD。
- 混合方案能保住玩家熟悉的判断链：敌方威胁在上、当前操作在下、资源靠近手牌；同时把问名、正名、三坛、裁定感集中到中央残榜案面。

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ 顶部窄条：执簿者 / 己形 / 墨 / 劫数 / 榜裂 / 设置                          │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│                    敌方压案区：敌人剪影 / 形条 / 名格                      │
│             当前行动：来势 / 异动 / 遮势       后一动小预告                │
│                                                                            │
│        左案角：短状态        残榜审案中轴        右案角：法宝挂位           │
│        己形 / 墨提示         问名格 / 正名印       器物 / 香封 / 反噬        │
│                              现形反扑短提示                                │
│                                                                            │
│          三坛时间轴：人坛       地坛       天坛                             │
│                                                                            │
├────────────────────────────────────────────────────────────────────────────┤
│ 操作上沿：香火 3/3 | 抽 12 | 弃 4 | 耗 0 | 目标：纸面鬼 | 结束回合         │
│ 手牌横排： [朱符] [问名] [护案符] [点额] [整卷]                            │
│ 左下短反馈：最近事件 1-2 条                      右下：日志 / 详情抽屉     │
└────────────────────────────────────────────────────────────────────────────┘
```

优点：

- 首战截图能立即看出敌人上、手牌下、当前行动在敌人旁。
- Boss 开场可通过上方敌方压案区放大体量，同时让中央残榜中轴承接“终审压案”的仪式感。
- 当前 `.battle-layout` 可以从双列改为单个战斗 shell：Phaser 案面做背景 / 战场层，React 战斗信息覆盖在同一视觉层。
- 日志自然降权，右侧长 HUD 不再主导画面。
- 法宝、香封和反噬以“案角器物”出现，不混入手牌，也不会在第一版形成过重侧栏。

风险：

- 需要把当前 `EnemyListPanel`、资源格、手牌和日志从纵向 HUD 重新拆到战斗页的固定区域。
- 1280x720 下卡牌横排空间紧，需要控制卡牌最小宽度、规则文本行数和 hover / tooltip。
- 左右案角只能放短状态和器物图标，不能把完整日志、长规则和大段资源说明重新塞回两侧。

T96 最小实现建议：

- 保留现有核心数据和组件逻辑，先重排 DOM / CSS，不改 battle resolver。
- 新增 active battle 专用布局容器，例如 `battle-screen-shell`、`battle-top-bar`、`battle-arena`、`battle-case-axis`、`battle-corner-state`、`battle-command-row`、`battle-hand-row`、`battle-log-drawer`。
- `EnemyListPanel` 改为 arena 内横排 / 网格敌方卡片；首战单敌居中，Boss 居中放大，多敌等宽排列。
- 残榜中轴固定承担问名格、正名印、现形反扑和三坛位置，避免这些本作核心信息散落到普通资源条里。
- `hand-list` 在 active battle 中改为横向卡牌轨道，移动端再折为水平滚动。
- `log-section` 默认折叠为最近两条，完整日志通过按钮展开。

### 备选方案：完整审案中轴 + 左右辅栏布局

适合后续正式视觉更强的版本，不建议 T96 第一版直接做满。

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ 顶部：本局状态 / 角色 / 风险 / 设置                                         │
├──────────────┬──────────────────────────────────────────────┬──────────────┤
│ 左辅栏       │ 中央审案区                                    │ 右辅栏       │
│ 己形 / 墨    │ 敌方压案                                      │ 法宝挂位     │
│ 抽弃耗       │ 来势 / 异动 / 名格                            │ 香封         │
│ 短反馈       │ 残榜中轴                                      │ 长期登簿     │
│              │ 三坛                                          │              │
├──────────────┴──────────────────────────────────────────────┴──────────────┤
│ 底部整宽手牌 / 香火 / 目标 / 结束回合                                      │
└────────────────────────────────────────────────────────────────────────────┘
```

优点：

- 桌案审判感更强，左 / 右辅栏可分别承担执簿者状态和器物状态。
- 法宝、香封、登簿、临诏等长期状态有稳定位置。

风险：

- 如果辅栏过宽，会重现“左右面板挤压战场”的问题。
- 当前实现里状态种类很多，第一版容易又变成两边都是文字表。
- 移动端和低视口成本更高。

适用条件：

- T96 先用推荐混合方案跑通后，若截图证明中央战场空间仍足够，再把法宝、香封、长期登簿和更多角色状态细分到左右辅栏。
- 如果低视口截图显示中央案面被辅栏挤压，应继续保持 T96 的轻案角，不升级为完整左右辅栏。

## 6. 国风视觉基调落点

本作的国风不应只表现为红金配色。首屏要落到以下具体对象：

- 战场不是“竞技场”，而是“残榜案面”：卷轴、木案、朱砂、墨迹、烧痕、印章形成战斗空间。
- 敌人不是普通怪物卡片，而是“压案敌形”：剪影压在榜页上，形条像被削薄的纸 / 煞气层。
- 来势不是普通攻击图标，而是“压案方向”：金褐或暗金冲击线，从敌人指向案面中线。
- 异动不是 debuff 图标，而是“朱红异兆”：印记、裂纸、污卷、遮名雾层，并明确提示需要断异动。
- 问名 / 正名在残榜中轴发生：名格像名籍格，正名像朱印落下，现形反扑像伪名剥落后露出更清楚的行动。
- 三坛不是普通 buff 槽，而是案面中线的三枚时间轴器位。
- 法宝不是卡牌，不进手牌；以右侧挂器、案角器物或残榜边缘小图标呈现。

颜色控制：

- 主基调用旧纸、木案、墨灰、青铜、朱砂、暗金。
- 避免大面积蓝紫仙侠渐变，也避免满屏红金导致所有危险都一样。
- 来势、异动、遮势、正名、法宝反噬要有可区分的形状语言，不只靠颜色。

## 7. T96 验收建议

T96 应做“可运行布局重构”，不是正式美术完工。

最低验收：

- 首战 1280x720 截图中，敌方、当前行动、香火、手牌、结束回合按钮同屏可见。
- Boss 开场截图中，Boss 位于中上方或中央压案区，来势 / 异动 / 遮势提示贴近 Boss，不再藏在右侧长 HUD。
- 裁定页截图能延续残榜 / 判印方向，登簿、朱批、削籍三类一眼可分。
- 日志默认降权，不再占据首屏主要列；最近关键事件仍可见。
- 推荐混合方案的 active battle 在 1280x720、1366x768、1920x1080 无水平溢出。
- 左右案角只承载短状态和器物挂位，不重新形成压缩中央案面的长 HUD。
- 不改核心战斗结算，不新增生产依赖，不新增正式素材。

建议截图清单：

- 标题页。
- 首战第一回合。
- 第二战带来势 / 异动提示。
- 多敌中段普通战。
- 窃榜使 Boss 开场。
- 归册后的裁定页。

建议验证：

- `npm run typecheck`
- `npm run lint`
- `npm run test`
- `npm run build`
- `git diff --check`
- 扩展或新增 T96 截图脚本，复用 T94 的三视口 manifest 检查。

## 8. 结论

T96 推荐先落“横向卡牌骨架 + 残榜审案中轴”的混合方案。

原因很具体：头部卡牌肉鸽的共同结构是“中部战场 + 底部手牌 + 边缘状态”，玩家看到截图时会自然知道这是一个可操作的卡牌战斗页；本作再把这个结构翻译成“残榜审案案面 + 敌方压案 + 三坛中轴 + 轻案角器物挂位”，就能同时保住类型可读性和国风题材识别。

当前右侧长 HUD 的问题不是信息不够，而是信息没有回到战斗对象上。T96 的核心不是加美术，而是把敌人、行动、手牌、资源和日志重新放回玩家决策发生的位置。
