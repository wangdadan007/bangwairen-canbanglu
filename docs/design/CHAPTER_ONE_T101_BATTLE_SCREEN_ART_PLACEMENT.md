# T101 战斗画面构图定稿 / 美术落位规格

日期：2026-05-27
任务：T101 战斗画面构图定稿 / 美术落位规格
性质：战斗首屏构图决策、正式美术落位规格、可运行战斗 shell 重排、AI 概念候选前置边界；不生成 AI 图

## 1. 目标

T101 解决 T99/T100 截图暴露的问题：当前画面信息点已能跑通，但框太多、主舞台不够简洁，T100 短动效在静态截图里也不容易看出来。后续如果直接画人物或敌人，很可能因为落位不清而返工。

本轮定稿的目标是：

- 战斗常态像成熟卡牌肉鸽一样清楚：左我、右敌、底部手牌、顶部资源。
- 中间不放常驻大面板，留作出牌、破形、问名、封势等动作通道。
- 残榜不做常驻中央信息框，只在背景、敌方贴身信息和关键反馈中出现。
- 直接把现有 active battle shell 重排为本规格的可运行版本，供后续截图审阅。
- 为后续 AI 概念候选、人工正式美术和特效资源提供落位规格。

## 2. 核心结论

新战斗首屏采用：

```text
顶部：战斗名 / 墨 / 劫数 / 榜裂 / 设置

主舞台：
左侧：执簿者形象 + 己形条 + 少量状态
中间：动作通道，用于出牌飞行、破形、问名、封势、正名反馈
右侧：敌人形象 + 敌形条 + 来势 / 异动 / 名格

底部：香火 / 手牌 / 临诏 / 法宝快捷 / 香封快捷 / 结束回合
```

明确不再采用：

- 不做常驻“中央残榜信息面板”。
- 不把执簿者锚点压在手牌左下作为底部附属状态。
- 不把敌方信息做成横向大表格。
- 不把顶部反馈卡作为 T100 动效的主要表现方式。

## 3. 竞品借鉴口径

本轮根据用户提供的《Slay the Spire》与《Monster Train》战斗截图修正判断。

可借鉴点：

- 敌我之间保留空场，作为动作、出牌和反馈通道。
- 玩家与敌人的核心状态贴近各自对象，而不是铺成说明面板。
- 手牌和费用在底部形成唯一主要操作层。
- 长期资源和设置放在顶部边缘，不能抢主舞台。
- 多信息并不等于多框；信息越多，越要贴对象、贴操作、贴触发点。

本作差异：

- 敌方右侧不仅显示敌形，还要显示名格、正名、来势、异动。
- 残榜审案感不能靠常驻大面板硬压在中间，而要靠背景纹理、名格 UI、问名 / 正名临时展开和裁定页主视觉形成记忆。
- 执簿者是榜外审案者，不是普通西式站桩英雄；左侧形象应像“案前执簿”，不是跳进中场挥剑。

## 4. 常态布局规格

### 4.1 顶部极简资源条

常驻内容：

- 第几战 / 遭遇名。
- 墨。
- 劫数。
- 榜裂。
- 设置 / 重开等系统入口。

不常驻：

- 长战斗解释。
- 反馈卡堆叠。
- 详细牌堆说明。
- 法宝完整规则文本。

顶部资源条要像竞品的状态边缘，不承担主叙事。

### 4.2 左侧执簿者区

用途：

- 承载衡简、照微、莲烬未来人物图或程序化占位。
- 显示己形条、角色名、少量当前状态。
- 显示起始法宝身份 cue，但不展开完整法宝规则。
- 显示三坛节奏状态：人坛、地坛、天坛贴在执簿者下方，表达“玩家正在布置的战斗时间轴”。

规格：

- 横向占主舞台宽度约 20%-26%。
- 角色形象朝右，视线或动作指向敌方。
- 己形条贴近角色脚下或身侧。
- 只保留少量状态小印记，例如当前目标、三坛、临时增益、反噬预警。
- 不显示手 / 抽 / 弃 / 耗完整统计；这些属于底部操作层。

后续人物图规格：

- 第一轮源图优先全身或近全身战斗 sprite，保留完整轮廓和足够留白。
- 战斗页实际展示时可按左侧主舞台裁切 / 缩放为近全身或 3/4 身，但不能损失身份物和主要姿态。
- 角色外轮廓必须在 1280x720 下仍可识别。
- 避免宽大袖袍或光效盖住中间动作通道。
- 衡简偏直立收束，照微偏窄身镜光，莲烬偏前倾火轮。

### 4.3 中间动作通道

用途：

- 出牌飞行。
- 破形轨迹。
- 问名墨迹。
- 封势压住来势。
- 正名判印。
- 法宝短触发。

规格：

- 常态保持空，不放大块 UI。
- 允许有很淡的桌面 / 残榜纹理，但透明度低，不能挡牌、敌人或反馈。
- 动效从玩家侧发出，穿过中间，落在敌方或敌方贴身信息上。
- 中间区域不承载常驻文字说明。
- 三坛和临诏不常驻在中间；三坛归左侧执簿者，临诏归底部快捷区。

### 4.4 右侧敌方区

用途：

- 承载纸面鬼、普通敌人、精英、窃榜使未来敌人图。
- 显示敌形条、敌名、来势 / 异动、名格、正名 / 现形状态。

规格：

- 单敌常态占主舞台宽度约 28%-36%。
- Boss 可扩展到 36%-44%，但不压过底部手牌。
- 敌方信息贴在敌人上方、下方或右侧，不做横向大表格。
- 敌形条贴敌人脚下或身侧。
- 名格以小卷轴 / 名札形式贴近敌人，问名后显墨，正名后落印。
- 来势与异动是两个不同标记：来势可被封势压住，异动必须有更强警示。

多敌规格：

- 多敌时仍在右侧 / 中右形成队列，不回到整屏横向面板。
- 当前目标放大或加印记，非目标降权。
- 每个敌人只显示简化敌形 / 意图 / 名格摘要，详细信息 hover 或展开。

### 4.5 底部操作层

常驻内容：

- 香火。
- 手牌横排。
- 临诏快捷状态。
- 法宝快捷入口。
- 香封快捷入口。
- 结束回合。
- 抽 / 弃 / 耗的轻量数量。

原则：

- 底部只服务操作，不再承载角色身份。
- 手牌是唯一主要操作层，不能被日志和资源框挤压。
- 临诏是本场持续规则槽，作为底部快捷状态显示，不进入中央动作通道。
- 法宝和香封以小挂件或快捷按钮存在，不混成普通手牌。
- 日志默认收起，只保留最近 1 到 2 条短反馈或抽屉入口。

## 5. 残榜呈现规则

### 5.1 背景化

主舞台地面 / 桌面可以有一条很淡的残榜卷轴纹理。

要求：

- 不在中间形成可读文字面板。
- 不抢玩家与敌人形象。
- 不影响卡牌飞行和反馈动效。

### 5.2 贴敌方信息

名格、正名、来势、异动这些“审案信息”贴近敌人，因为被审对象是敌人。

落位：

- 名格：敌人下方或侧边名札。
- 正名：名格区域盖印，敌人轮廓有现形变化。
- 来势：敌形条附近的可封标记。
- 异动：敌人上方或侧边的朱红警示标记。
- 后一动：当前行动下方的小预告，视觉权重低于当前行动。

### 5.3 关键时临时展开

残榜只在关键节点短暂展开：

- 问名：名格处有小卷轴 / 名札展开，墨迹显名。
- 正名：敌方名格区域落朱印，敌人伪名层剥落。
- 归册 / 裁定：进入裁定页时残榜成为主视觉。
- 削籍：刮纸 / 裂榜短反馈只在裁定选择时出现。

常态结束后收回，不常驻挡场。

## 6. T100 动效重定向

T100 后续应从“顶部反馈卡闪一下”转向“动作发生在对应位置”。

| 反馈 | 新落点 | 表现方向 |
|---|---|---|
| 破形 | 敌人本体 / 敌形条 | 朱砂裂痕、敌形条震动、纸面撕裂 |
| 问名 | 敌方名格 / 名札 | 墨迹显名、小卷展开 |
| 正名 | 敌方名格 + 敌人轮廓 | 判印落定、伪名层剥落、现形反扑提示 |
| 封势 | 来势标记 | 符纸压住、来势箭头被截断 |
| 异动 | 敌人侧警示 | 朱红警示纹、异动源头冒出 |
| 裁定 | 裁定页残榜 | 榜面展开、三判印落位 |

保留原则：

- 仍受动画开关和 `prefers-reduced-motion` 约束。
- 关闭动画时，文字和状态标记仍能说明结果。
- 动效持续应足够被截图捕捉，建议关键状态保留 0.5 到 1 秒可见残留。

## 7. 后续资产规格

### 7.1 执簿者角色图

建议规格：

- 源图优先全身或近全身战斗 sprite，完整轮廓、脚部和身份物都要保留。
- 战斗页可按左侧主舞台缩放或少量裁切为近全身 / 3/4 身显示。
- 二轮精修或接入切片时再处理透明底。
- 朝右。
- 可裁切到 1280x720 左侧主舞台仍清楚。
- 每名角色必须有一个可缩成小图标的身份物件：
  - 衡简：打神鞭残节 / 旧榜笔。
  - 照微：照骨镜。
  - 莲烬：赤绫火轮。

禁区：

- 不做可收集神将立绘感。
- 不直接复刻姜子牙、杨戬、哪吒等原典角色。
- 不用大面积蓝紫仙侠光效。

### 7.2 敌人图

建议规格：

- 透明底或少背景半身 / 全身。
- 朝左或面向中间。
- 普通敌人可轻量，Boss 需要更高体量。
- 纸面鬼必须轻、薄、纸页化。
- 窃榜使必须比普通敌人有明显压迫感，但不能像可招募神将。

### 7.3 UI 与特效素材

优先级：

1. 名格 / 名札 UI。
2. 来势 / 异动标记。
3. 破形、问名、正名、封势短特效。
4. 裁定页残榜 / 判印。
5. 临诏 / 法宝 / 香封快捷图标。

所有 AI 概念图或特效参考图必须先登记为“概念候选 / 不入正式构建”，不默认可商用。

## 8. T101 可运行重排落地

用户确认后，T101 从纯规格任务扩展为“规格 + 可运行重排”。

已落地范围：

- active battle shell 改为左执簿者 / 中动作通道 / 右敌方。
- 三坛从敌我之间移到左侧执簿者信息下方；临诏从敌我之间移到底部快捷区。
- 底部手牌区移除角色身份负担，只保留香火、手牌、临诏、法宝 / 香封快捷、结束回合和轻量日志。
- 敌方信息从横向表格改成贴敌方形象的小标记。
- 残榜中轴大面板降为背景纹理和临时反馈。
- T100 反馈落点改到敌人、名格、来势 / 异动和裁定页；反馈文字降到中央动作通道的小型短反馈。
- 新增 `capture:t101` 截图脚本，覆盖首战、第二战、多敌、Boss、裁定。

本轮不做：

- AI 生图。
- 正式人物图入库。
- 新玩法。
- 移动端完整重排。

## 9. 验收

- 任何截图第一眼都能看出：左侧是执簿者，右侧是敌人，底部是手牌操作层。
- 中间没有常驻大信息面板，能作为出牌和反馈通道。
- 中间不常驻三坛或临诏；三坛贴左侧执簿者，临诏贴底部快捷。
- 残榜审案感仍存在，但通过背景、敌方名格、关键反馈和裁定页表达。
- 顶栏资源极简，不再堆反馈卡和解释文本。
- 底栏不承载角色身份，手牌成为唯一主要操作区。
- 后续 AI / 人工资产可以按本规格生产，不需要先猜角色和敌人该放哪里。

## 10. 剩余风险

- T101 已包含可运行 UI 重排，但仍是程序化占位，不是正式美术定稿。
- 如果后续正式角色图体量过大，仍可能需要微调左侧宽度和中间动作通道。
- 多敌战在右侧队列中的可读性需要继续人工审图；本轮先解决首屏结构，不解决全部多敌排布细节。
- PRD-pages 已随本轮同步修正战斗页口径，但旧 T95/T96 文档仍记录历史方案，后续以 T101 为最新执行规格。
