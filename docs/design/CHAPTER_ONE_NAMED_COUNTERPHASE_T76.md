# T76 正名后剥诡计与现形反扑设计方案

日期：2026-05-23
适用范围：第一章所有当前可正名敌人

## 1. 设计结论

T76 先把设计口径定清楚：正名后不应只是让敌人单纯削弱，也不应让玩家花资源问名后只换来更强惩罚。当前实现已把本方案落到逐敌人 `onNamed` / `namedPhase` 数据、battle resolver、HUD / actionLog 和测试中。

第一章采用“剥诡计 + 现形反扑”结构：

1. 正名前，敌人主要靠伪名、遮名、偷香、污卷、召唤、挪坛、遮势等方式扰乱玩家。
2. 正名瞬间，触发名破，削减敌人形，并进入现形阶段。
3. 正名后，敌人的伪装、逃名或干扰类机制被禁用、降级或转化；同时敌人改用更清楚、更直接、更可预警的来势或终段压力反扑。

玩家获得的确定收益是：名破、归册裁定入口、遮名不能回盖名格，以及逐敌人伪计被剥除。敌人保留的高潮压力是：现形后的直接来势、可全封压住的附带后果，或 Boss 终审压案。

## 2. 规则边界

- 正名不是第二套胜利条件；形归零仍是唯一基础胜利条件。
- 正名不是无条件把敌人打残废；它应让敌人“不能继续用假名和诡计逃”，但仍能正面反扑。
- 普通敌人不强求每只都有复杂二阶段；教学怪和纯来势怪可以只获得名破和日志反馈。
- 精英和 Boss 必须有明确的正名前后差异，但不新增第 4 个精英、第二个 Boss 或完整多阶段演出。
- 正名后禁用或降级的主要是异动、遮名、逃名、牌堆污染、召唤、坛位干扰和信息压迫；不要把封势变成万能防御。
- 现形反扑应优先使用来势、明牌预警、未全封后果和已有路线压力字段，不做无法预警或无法反制的随机惩罚。

## 3. 数据化建议

实现时不建议继续只靠散落在 resolver 里的条件判断。当前采用 `EnemyDefinition.onNamed` / `namedPhase` 数据化表达三块：

| 字段 | 含义 |
|---|---|
| `severedMechanics` | 正名后禁用、降级或转化的伪计，例如遮名、偷香、塞污卷、召唤、挪坛。 |
| `revealedCounterIntent` | 正名后优先切入或提高权重的现形反扑，例如来势、终审拍案、火压、震坛。 |
| `logKey` / `hintKey` | HUD 和 actionLog 文案，说明“剥掉了什么，接下来会怎么反扑”。 |

当前实现采用最小数据化结构：正名后根据敌人 `onNamed` 写入并激活 `namedPhase` 状态，敌人结算异动、来势后果和推进意图时读取该状态；不需要完整脚本系统。

## 4. 全可正名敌人设计表

当前 `src/data/enemies.json` 中 `nameSlots > 0` 的敌人共 13 个。无名纸祟、香灰虱和供桌残像不在本表范围。

| 敌人 | 正名前压力 | 正名后剥掉的诡计 | 现形反扑 | 设计目的 |
|---|---|---|---|---|
| 纸面鬼 | 基础来势，教学形、问名、名破 | 无专属诡计，只记录“纸名已明” | 维持纸刃扑案，不额外强化 | 首战基准：让玩家理解正名和名破，不引入二阶段负担。 |
| 偷香鼠 | 偷香异动，下回合香火 -1 | 正名后偷香失败或转化为低来势，不再扣香火 | 鼠齿扑案成为主要压力，可小幅提高一次来势 | 玩家学到“查明它就抓不住香火”，但仍要处理正面攻击。 |
| 铜铃夜巡 | 稳定来势，封势教学 | 无伪计可剥，只暴露其巡铃节奏 | 铃声逼案保持明牌来势，可在日志提示“只剩正面压案” | 纯来势敌的基准：正名收益主要是名破和归册。 |
| 无灯庙祝 | 召唤无名纸祟，多敌拖节奏 | 正名后不再新增召唤；已在场召唤物不消失 | 磬声逼案或扫灰压案成为主压力，若纸祟仍在可保留轻微来势加压 | 剥掉支援展开，留下清场和封势选择。 |
| 断签客 | 倒签入账，短期收益换劫数 | 正名后不能再把劫数伪装成收益；倒签入账失效或转成明示来势 | 断签逼债 / 翻兆逼案作为直接来势，可读取已入账劫数轻微加压 | 把“诱惑交易”从模糊账目改成明牌债压。 |
| 灰坛童子 | 撒灰乱供，偷香或压地坛窗口 | 正名后撒灰乱供不再偷香，或只给弱提示 / 弱来势 | 灰盂压案读取地坛空置状态，仍可被封势处理 | 剥掉乱供偷香，保留地坛时间点考核。 |
| 瘟纸人 | 瘟墨染卷，向牌堆慢性污染 | 正名后不再新增污卷，或下一次染卷失效 | 瘟灰扑案成为主要压力；若牌堆已有污卷，可有轻微明示来势加压 | 玩家用正名截断污染源，但仍要处理已污染牌堆和直接来势。 |
| 塞卷吏 | 塞入污卷，污染更快显现 | 正名后不能再把污卷塞到抽牌堆顶；当前实现改为塞入弃牌堆 | 卷匣拍案读取已有污卷压力，未全封时才触发轻后果 | 剥掉最烦的牌堆偷袭，保留“污卷越积越压案”的节奏。 |
| 逃名纸马 | 遮名、逃名，拖慢归册 | 正名后遮名彻底失效，不能回盖名格 | 纸蹄扬尘作为高但明牌的来势收束 | 正名直接克制逃名主题，代价是承受现形冲撞。 |
| 窃香吏 | 查香簿 / 勾验供香，首次偷香更重 | 正名后查香簿失效或偷香量降为 0；香火账不再继续扩大 | 吏印拍案或举炉压案成为税审反扑，可读取既有香火账小幅加压 | 精英收益明确：剥掉资源审查，但进入正面压案。 |
| 火部逃名者 | 焚名火遮名，正名前惩罚问名拖延 | 正名后不能再焚名回盖名格 | 裂牌火势 / 火印压案转为明牌火压，可提高来势但能被封势处理 | 从“护名”切到“抗火”，形成清晰中后段高潮。 |
| 斗部空壳 | 空壳遮名、挪坛或干扰坛位 | 正名后不再遮名；挪坛从取消收益降级为可全封压住的震坛后果 | 坠星拍案作为现形反扑，读取已布置坛位但必须预警 | 剥掉空名和暗挪坛，保留三坛时机考核。 |
| 窃榜使 | 偷香、遮名、塞污卷、遮势和路线压力组合 | 正名后固定禁用遮取空名；再按路线压力剥掉一类主诡计：稳行优先偷香，归册优先遮势，高榜裂优先塞污卷 | 终审拍案进入终段主压力；未全封时根据路线产生可预警后果 | Boss 不塌场：玩家剥掉窃榜伪计，但迎来第一章终审。 |

## 5. 实现优先级与当前状态

| 优先级 | 范围 | 原因 |
|---|---|---|
| P0 | 逃名纸马、火部逃名者、斗部空壳、窃榜使 | 它们直接涉及遮名、逃名、坛位和 Boss 综合考核，最能体现正名前后差异。 |
| P1 | 偷香鼠、无灯庙祝、断签客、瘟纸人、塞卷吏、窃香吏 | 它们覆盖偷香、召唤、劫数、污卷和资源审查，适合第二批数据化。 |
| P2 | 纸面鬼、铜铃夜巡、灰坛童子 | 教学或轻机制敌，只给日志和轻量现形反扑，不做复杂二阶段。 |

当前开发状态：

- 13 个可正名敌人均已写入 `onNamed` 数据。
- 正名后已支持禁用异动、降级异动、按路线禁用 / 降级异动、按路线禁用来势后果和优先切入现形反扑意图。
- 窃榜使已按 T76 方案补齐三路线：稳行线偷供失准，归册线终审遮势失效，高榜裂线撕榜塞卷和终审污卷失效。

## 6. UI 与文案口径

玩家可见表达建议统一为：

- “真名已明：伪计退散。”
- “现形反扑：敌人改用明牌来势。”
- “遮名失效：真名已定，不能再盖回名格。”
- “终审将至：窃榜伪计已破，下一次压案更直接。”

不要写成“敌人虚弱”或“机制解除完毕”。更准确的玩家理解是：我查明了它，它不能再赖名逃账；但它现形后会正面拼命。

## 7. 不做内容

- 不新增敌人、Boss、卡牌、法宝、事件、遭遇模板或生产依赖。
- 不改变伏诛 / 归册奖励同池 3 选 1。
- 不改变形归零胜利条件、裁定三类、来势 / 异动边界、三坛规则和法宝不入牌堆。
- 不把所有普通敌人都做成复杂二阶段；普通战仍要保持 3 到 6 回合目标节奏。
