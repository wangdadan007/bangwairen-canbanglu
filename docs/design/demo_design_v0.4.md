v0.5 在 v0.4 基础上补充以下口径修订：

- 构筑流派不再表述为五个完全平行、同等成熟的独立流派。第一章当前采用“破形直攻 / 正名裁定”两条主轴，叠加朱批改牌、封势断异、裂榜逆命等构筑模块。

- “劫数逆命流”改称“裂榜逆命流”，当前定位为高风险爆发层，而不是完全独立成熟主流派。

v0.4 在 v0.3 基础上补充以下定案：

## 1. 项目定位

《榜外人：残榜录》是一款封神题材单机牌组构筑肉鸽。玩家不是姜子牙、哪吒、杨戬，也不是商周阵营角色，而是封神榜外的执簿者。封神大战之后，封神榜出现裂痕，玩家进入残榜，通过战斗、问名、正名和裁定重新整理名籍。

本作不做传统封神神将收集，不做敌魂变卡，不做“封神皮肤的普通卡牌肉鸽”。核心体验是：玩家在一场场战斗中查名、破形、正名、归册、登簿、朱批、削籍，逐步改写一卷破裂的封神榜。

## 2. 核心定论

- 唯一基础胜利条件：把敌人的“形”打空。
- “破形”是所有流派最终必须服务的终局轴，但“破形直攻流”只是其中一种进攻流派。
- “名”是敌人的真实身份、罪状、归属、神职空缺或残榜记录。
- 问名揭示敌人的名。名全部揭示后，敌人进入正名。
- 正名触发“名破”，使敌人的形大幅降低；设计口径为剥掉敌人的伪名 / 逃名 / 干扰类诡计，并让敌人进入更清楚、更可预警的现形反扑。
- 未正名时打空形，结算为“伏诛”，战后卡牌奖励与归册同池三选一，节奏更快。
- 正名后打空形，结算为“归册”，先进入裁定，再领取同池卡牌奖励。
- 裁定只保留三类：登簿、朱批、削籍。
- 不加入留魂，不加入留名，不加入敌魂变卡，不加入 Boss 饶恕后助战。
- 敌方行动分为来势和异动。封势只能处理来势，异动需要专门牌。
- 不是所有普通敌人都需要完整问名和正名。普通战要保持节奏，精英和 Boss 才是完整问名/正名/裁定的主要承载对象。

## 3. 形与名

“形”是敌人的战斗实体，可理解为形骸、煞气、法力外壳或显化之身。玩家通过破形牌、法宝、朱批、名破、火印、雷符、鞭击、反势等方式削减敌人的形。形归零时，敌人退出战斗。

“名”是敌人的真实结构，可以包含来历、罪状、神职空缺、榜册状态等信息。玩家通过问名、照骨、验符、卜辞等牌揭示敌人的名。名全部揭示后，敌人进入正名。

胜利结算：

| 条件 | 结算 | 奖励 |
|---|---|---|
| 形归零但未正名 | 伏诛 | 与归册共用同一战后卡牌奖励池，3 选 1 |
| 形归零且已正名 | 归册 | 先进入裁定，再领取同池卡牌奖励 |

归册线不作为基础胜利条件，只作为后期卡牌、法宝或登簿条目的高级能力。

## 4. 敌人分层

| 敌人层级 | 是否有名 | 是否有正名 | 是否有裁定 | 作用 |
|---|---|---|---|---|
| 无名杂祟 | 无 | 无 | 无 | 快速战斗、试牌、清场、保持节奏 |
| 有名普通敌人 | 1-2 格 | 可选 | 小奖励或弱裁定 | 普通战里的轻度策略 |
| 关键普通敌人 | 2 格 | 可选 | 简化裁定 | 事件战或章节特色敌人 |
| 精英敌人 | 3 格 | 可选但强收益 | 完整裁定 | 高风险高收益 |
| Boss | 多段或多阶段 | 强烈推荐 | 章节级裁定 | 机制总考 |

## 5. 来势、异动与封势

来势是敌人的直接冲击行为，例如扑案、刀煞、爪击、雷击。来势可以被封势降低。

异动是非直接攻击的特殊行为，例如偷香、塞污卷、治疗、召唤、遮名、祛印、偷关键词、改变手牌。普通封势不能处理异动，必须使用对应卡牌、法宝、坛位或条件处理。

设计边界：封势不是格挡，不是万能防御。封势只处理“来势”，不处理“异动”。这一点必须在战斗逻辑中写死。

## 6. 基础卡牌语法

| 语法 | 含义 |
|---|---|
| 破形 X | 削减敌人的形 |
| 问名 X | 揭示目标 X 格名 |
| 封势 X | 降低敌人直接来势 X |
| 断异动 | 阻止或削弱某类异动 |
| 正名 | 目标所有名格被揭示，触发名破 |
| 朱批 | 对已有卡牌添加永久改造词条 |
| 登簿 | 把条目写入榜格，成为本局永久规则 |
| 削籍 | 抹去名籍获得强收益，但增加榜裂 |
| 临诏 | 打出后挂入临诏区，本场战斗持续生效，战后消失 |

首次出现专有名词时，UI 使用“白话 + 术语”的写法：敌人的形体（形）、击破敌形（破形）、查明真名（问名）、真名已明（正名）、战后改榜（裁定）、写入榜册（登簿）、批改卡牌（朱批）、抹去名籍（削籍）。

## 8. 流派结构

破形是终局轴，但不是唯一玩法。第一章当前不把五个方向都当作完全平行、同等成熟的独立流派，而是按“两条主轴 + 构筑模块”理解：破形直攻和正名裁定是主要推进轴；朱批改牌、封势断异、裂榜逆命分别提供引擎、防守壳和风险爆发层。非直攻构筑必须有破形转换器。

| 方向 | 当前定位 | 如何打空形 |
|---|---|---|
| 破形直攻流 | 成熟主轴 | 雷符、火符、鞭击、阵法、判印 / 金印副模块等直接破形。 |
| 正名裁定流 | 成熟主轴 | 问名、照骨、名破和裁定收益推动归册；归册不独占卡牌奖励，核心价值来自名破、剥诡计 / 现形反扑后的可控性和裁定。 |
| 朱批改牌流 | 构筑引擎 | 朱批赋予返香火、抽牌、额外问名、条件破形、三坛或风险收益；不承诺无限循环或复制批注。 |
| 封势断异控场流 | 防守与反势外壳 | 封势处理来势，断异动 / 地坛 / 专门牌处理异动，再转成破形、问名或朱批收益。 |
| 裂榜逆命流 | 风险爆发层 | 用劫数、榜裂、削籍短效增强、借劫和高风险破形换即时爆发。 |

### 8.1 临诏试点

临诏是一种可进入牌组的战斗内持续规则牌。打出后不进弃牌堆，而是挂入“临诏区”；本场战斗持续生效，战后消失。

边界：

- 临诏负责本场战斗持续规则。
- 朱批负责永久改造某张卡。
- 登簿负责本局永久规则。
- 法宝负责牌组外器物成长。
- 三坛负责有明确触发窗口的一次性战斗时间轴状态。

第一版临诏区上限为 2，同名不叠加；所有打出、入区、触发、上限失败和战后清除都写入 actionLog。第一批不扩卡池，已从现有敕令 / 支援牌改造明烛临诏、压煞临诏、朱痕临诏，围绕问名、封势 / 断异动和朱批做试点。临诏牌可被朱批，已挂入临诏区的状态不可作为朱批目标。

## 9. 破形直攻流的卡牌家族

- 雷符系：爆发破形。
- 火符系：持续破形，通过火印削形。
- 鞭击 / 打神鞭系：正名后强化破形。
- 金印 / 重敕系：高费用重击。
- 阵法 / 三坛合击系：依赖坛位布置。
- 反势系：把来势或异动反制转换成破形。
- 名破增幅系：属于正名裁定流的转换器，不是独立直攻家族。

## 10. 法宝系统定案

法宝不是卡牌本体。法宝是牌组外的局内器物，获得后显示在战斗界面榜册边缘。法宝不占手牌，不进入抽牌堆，不进入弃牌堆，不参与普通卡牌奖励池。

法宝可以影响卡牌，可以监听战斗事件，可以生成临时法宝牌或敕令牌，也可以在裁定界面改变选项。但“法宝本体”不洗入牌组。

法宝状态：

| 状态 | 含义 |
|---|---|
| 未认主 | 获得后基础效果，通常较弱或有条件 |
| 认主 | 满足条件后升级，成为构筑支点 |
| 反噬 | 过度使用、违背法宝条件或连续触发过载后产生一次性代价 |

Demo 法宝模板分三类：

| 类型 | 说明 | 示例 |
|---|---|---|
| 战斗触发型 | 每场首次问名、正名、归册、完封、朱批时触发 | 照骨镜、打神鞭残节 |
| 主动充能型 | 每场战斗开始获得充能，玩家点击法宝发动 | 朱砂斗 |
| 裁定影响型 | 只在归册后的裁定界面起作用 | 裂榜针 |

反噬窗口定案：

“连续两场战斗”不是反噬持续时间，而是反噬判定窗口。若同一件法宝连续两场触发过载，第二场结束后进入反噬；反噬通常只在下一场战斗生效一次，生效后清除。

示例：

| 法宝 | 未认主 | 认主条件 | 认主 | 过载 | 反噬 |
|---|---|---|---|---|---|
| 打神鞭残节 | 每场首次正名后，本回合下一张破形牌额外破形 2 | 本局累计归册 3 个有名敌人 | 每场首次正名后，本回合下一张破形牌额外破形 5 | 未正名就借鞭势打空敌形，记录 1 次“妄鞭” | 若连续两场记录“妄鞭”，下一场战斗开始时，将 1 张“裂鞭残响”加入手牌 |
| 照骨镜 | 每场首次问名后，查看目标下一次行动 | 本局累计问名 8 次 | 每场首次问名后，查看目标下一次行动，并使下一次问名消耗 -1 | 对无名目标连续使用镜照 | 下一场首次问名前，目标行动意图被遮蔽 |
| 朱砂斗 | 每场 1 充能：本回合下一次朱批额外添加 1 个弱词条 | 本局累计朱批 3 次 | 每场 1 充能：本回合下一次朱批添加 1 个中等词条 | 对同一张牌连续叠批过多 | 下一场战斗开始时，随机 1 张手牌带“墨滞”：本回合无法被朱批 |
| 裂榜针 | 削籍收益提高 | 本局削籍 2 次 | 削籍收益大幅提高 | 连续两次削籍 | 下一场战斗开始时榜裂临时 +1，战后恢复 |

## 11. 第一章 Demo 内容规模

第一章名为“残榜初裂”。目标时长：单局约 30 到 45 分钟。

本章分为两个口径：MVP 口径用于内部开发最小闭环；公开 Demo 口径用于对外测试或 Steam Demo。

| 模块 | MVP 口径 | 公开 Demo 目标 |
|---|---:|---:|
| 可玩角色 | 1，执簿者 | 1，执簿者 |
| 卡牌 | 约 40 张 | 42 到 48 张 |
| 普通敌人单位 | 8 个 | 12 个 |
| 低成本杂祟 / 召唤物 | 可先不单列 | 4 个 |
| 普通遭遇模板 | 10 到 12 个 | 18 到 22 个 |
| 精英敌人 | 3 个 | 3 个 |
| 事件精英 | 可不做 | 可选 1 个 |
| Boss | 1 个，窃榜使 | 1 个，窃榜使 |
| Boss 行为变体 | 1 套 | 2 到 3 套 |
| 法宝 | 8 件 | 10 到 12 件 |
| 事件 | 6 个 | 8 到 10 个 |
| 商店 / 休整 / 路线 / 结算 | 必须 | 必须 |
| 存档与继续游戏 | 可延后到 Demo MVP | 公开 Demo 必须 |

结论：第一章 MVP 可以先按 8 普通、3 精英、1 Boss、8 法宝、6 事件推进；但公开 Demo 应扩展到更像真正肉鸽卡牌的遭遇池，不能只做线性 10 场固定战斗。

## 12. 第一章普通敌人与遭遇池

普通敌人单位建议扩展到 12 个，其中 3 到 4 个可以是低复杂度、低美术成本单位。

| 敌人 | 定位 | 主要机制 |
|---|---|---|
| 纸面鬼 | 教学单怪 | 形、名、问名、正名 |
| 无名纸祟 | 弱杂祟 | 无名、低形、多敌组合填充 |
| 偷香鼠 | 资源干扰 | 偷香、异动、断供教学 |
| 香灰虱 | 弱群怪 | 群体压力、配合偷香鼠 |
| 铜铃夜巡 | 来势教学 | 稳定来势、封势教学 |
| 无灯庙祝 | 支援怪 | 祈愿、召唤、强化杂祟 |
| 断签客 | 劫数教学 | 给玩家短期诱惑，埋劫数 |
| 灰坛童子 | 地坛教学 | 敌行动前异动、坛位干扰 |
| 瘟纸人 | 污染牌堆 | 塞污卷、反噬教学 |
| 塞卷吏 | 牌堆污染 | 把污卷、残页塞进抽牌堆 |
| 逃名纸马 | 遮名 / 逃亡 | 已问名后可能遮名或逃走 |
| 供桌残像 | 支援物 / 半敌人 | 治疗、加香、护住主怪 |

普通遭遇模板目标为 18 到 22 个。示例：

| 阶段 | 遭遇 | 目的 |
|---|---|---|
| 前段 | 纸面鬼 | 单怪教学 |
| 前段 | 纸面鬼 + 无名纸祟 | 教多目标但不复杂 |
| 前段 | 偷香鼠 | 教异动 |
| 前段 | 偷香鼠 + 香灰虱 | 教主怪 + 杂祟 |
| 前段 | 铜铃夜巡 | 教来势 / 封势 |
| 中段 | 无灯庙祝 + 无名纸祟 | 教召唤 / 支援 |
| 中段 | 断签客 | 教劫数诱惑 |
| 中段 | 塞卷吏 + 纸面鬼 | 教污卷干扰 |
| 中段 | 逃名纸马 + 香灰虱 | 教遮名和收尾压力 |
| 中段 | 灰坛童子 + 铜铃夜巡 | 教地坛时间点 |
| 后段 | 瘟纸人 | 教反噬 / 污染 |
| 后段 | 瘟纸人 + 塞卷吏 | 牌堆污染压力 |
| 后段 | 供桌残像 + 偷香鼠 | 支援怪保护异动怪 |
| 后段 | 灰坛童子 + 逃名纸马 | 问名与时机压力 |
| 后段 | 无灯庙祝 + 瘟纸人 | 召唤与污卷混合 |

## 13. 第一章精英与 Boss

精英数量保持 3 个，优先做扎实，不急着加数量。

| 精英 | 功能 |
|---|---|
| 窃香吏 | 解锁墨，考验偷香、断供、问名收益 |
| 火部逃名者 | 解锁削籍 / 榜裂，考验正名前后的机制差异 |
| 斗部空壳 | 解锁天坛 / 法宝认主，考验三坛和法宝联动 |

Boss 保持 1 个原创 Boss：窃榜使。不要第一章做多个 Boss。更好的方式是让窃榜使拥有 2 到 3 个行为变体：

| 变体 | 触发倾向 | 行为重点 |
|---|---|---|
| 低榜裂路线 | 玩家较少削籍、榜裂较低 | 偷名、遮名、伪归册 |
| 高榜裂路线 | 玩家频繁削籍、榜裂较高 | 撕榜、塞污卷、强化反噬 |
| 高归册路线 | 玩家归册数较高 | 针对已登簿 / 已朱批内容做反制 |

## 14. 第一章机制解锁

第一章仍采用逐战斗机制解锁，但从第 4 场后允许在已解锁机制范围内使用随机遭遇模板。前 3 场应尽量固定，保证玩家理解基本规则。

| 战斗 | 教学敌人 / 节点 | 新增内容 |
|---|---|---|
| 1 | 纸面鬼 | 形、破形、问名、正名、名破、伏诛/归册 |
| 2 | 偷香鼠 | 来势、异动、封势、断供 |
| 3 | 铜铃夜巡 | 朱批 |
| 4 | 无灯庙祝 | 奉坛、人坛 |
| 5 | 窃香吏（精英） | 墨 |
| 6 | 断签客 | 劫数 |
| 7 | 火部逃名者（精英） | 削籍、榜裂 |
| 8 | 灰坛童子 | 地坛 |
| 9 | 斗部空壳（精英） | 天坛、法宝认主 |
| 10 | 瘟纸人 | 反噬 |
| Boss | 窃榜使 | 综合考核 |

所有卡牌奖励池必须跟随机制解锁。未解锁机制不得出现在奖励、商店、事件、法宝描述中。

## 15. 正式版章节规划

正式版内部暂定为 4 个常规章 + 1 个终审短章，共 5 章。对外宣传阶段暂不承诺完整章数，避免过早锁死产能。

| 章 | 名称 | 功能 |
|---|---|---|
| 第一章 | 残榜初裂 | 核心教学：形、名、问名、正名、伏诛、归册、朱批、三坛、法宝 |
| 第二章 | 诸司错位 | 神职错乱，登簿深化，墨资源和榜格规则扩展 |
| 第三章 | 香火逆流 | 人间庙宇、香火、愿鬼、偷香、异动压力 |
| 第四章 | 万名相噬 | 榜裂后果、削籍诱惑、法宝反噬、敌人篡改名籍 |
| 第五章 | 封榜终审 | 短终局章，根据本局登簿、朱批、削籍、榜裂生成最终审判 |

第五章可以是 3 到 5 个节点 + 最终 Boss，不需要做成完整大地图。这样既能让正式版有完整结构，又不让开发量膨胀失控。

## 16. MVP 与开发阶段定义

MVP 不等于一开始完整做第一章。开发上拆成三层：

| 阶段 | 名称 | 目标 |
|---|---|---|
| 阶段 1 | 核心战斗 MVP | 1 个敌人、10 张左右卡、无路线、无商店、无完整法宝，只验证破形、问名、正名、伏诛 / 归册 |
| 阶段 2 | 教学纵切 MVP | 3 到 5 场连续战斗，有奖励、有前几次机制解锁、有朱批雏形 |
| 阶段 3 | 第一章 Demo MVP | 完整第一章，含路线、商店、休整、事件、精英、Boss、法宝、结算 |

开发优先级：先保证核心规则可玩、可测试、可复盘，再扩内容量和表现。

## 17. 推荐开发顺序

1. 技术骨架：TypeScript、Vite、React、Phaser、数据加载、战斗状态机。
2. 基础战斗循环：抽牌、费用、出牌、弃牌、回合结束、敌人行动。
3. 形与破形：敌人形、破形结算、形归零、胜负判断。
4. 来势、封势、异动：先把“封势不能处理异动”这个边界写死。
5. 名、问名、正名、名破：实现名格、揭示、正名触发、名破削形。
6. 伏诛 / 归册结算：同样形归零，卡牌奖励同池；归册额外进入裁定并保留名破与正名前后阶段变化收益。
7. 第一批 12 到 15 张卡 + 前 3 场教学战。
8. 裁定 MVP：先做朱批，再做登簿，最后做削籍 / 榜裂。
9. 路线地图、奖励、商店、休整、事件。
10. 墨、劫数、榜裂。
11. 三坛：先人坛，再地坛，再天坛。
12. 法宝基础效果、认主、反噬：先做未认主和认主，再做反噬。
13. 精英与 Boss。
14. 存档、继续游戏、设置、结算页。
15. 美术 / 音效占位与动效手感贯穿开发，最后阶段替换和强化。
16. 桌面打包与 Steam Demo 准备。

## 18. 技术结构建议

建议技术栈：TypeScript、Vite、React、Phaser 4.1.x，桌面包可先采用 Electron，后续视包体和性能再评估 Tauri。

架构原则：

- 战斗逻辑和渲染逻辑分离。
- 卡牌、敌人、法宝、事件、裁定尽量数据驱动。
- 不把大量玩法逻辑写死在 UI 组件里。
- 每张卡、每个敌人、每件法宝、每个事件都必须有唯一 ID。
- 状态变化要可追踪、可测试、可重放。
- 每次出牌、问名、正名、破形、封势、异动、裁定都应进入战斗日志。

核心数据文件建议：

```text
src/data/cards.json
src/data/enemies.json
src/data/encounters.json
src/data/artifacts.json
src/data/verdicts.json
src/data/events.json
src/data/routes.json
src/data/tutorial_unlocks.json
src/localization/zh-CN.json
src/localization/en-US.json
```

核心代码目录建议：

```text
src/game/core/        # 纯逻辑：状态机、规则、结算
src/game/data/        # 数据加载与校验
src/game/effects/     # 卡牌、敌人、法宝效果执行器
src/game/scenes/      # Phaser 场景
src/game/ui/          # React UI 组件
src/game/types/       # TypeScript 类型
src/game/tests/       # 规则测试
```

## 19. 首批 Codex 开发提示词

```text
你正在开发《榜外人：残榜录》，一款封神题材单机牌组构筑肉鸽。当前目标不是完成完整第一章，而是搭建“核心战斗 MVP”。

请先建立 TypeScript + Vite + React + Phaser 的项目骨架，并实现可运行的最小战斗场景。项目必须数据驱动，战斗逻辑和渲染逻辑分离。

核心规则：敌人有“形”和“名”。形归零是唯一基础胜利条件。玩家通过卡牌“破形 X”降低敌人的形。玩家可以通过“问名 X”揭示敌人的名。敌人全部名格被揭示后进入“正名”，正名触发“名破”，削减敌人一段形。未正名时形归零为“伏诛”，正名后形归零为“归册”。

本任务只做核心战斗 MVP：
1. 建立项目结构。
2. 建立 Card、Enemy、CombatState、PlayerState、ActionLog 等 TypeScript 类型。
3. 建立 cards.json 和 enemies.json，至少放入 6 张初始牌和 1 个敌人“纸面鬼”。
4. 实现抽牌、出牌、费用、弃牌、回合结束、敌人简单行动。
5. 实现破形、问名、正名、名破、伏诛/归册结算。
6. 实现简单 UI：玩家手牌、香火、敌人形、敌人名格、战斗日志、结束回合按钮。
7. 编写基础测试，覆盖破形、问名、正名、形归零结算。

不要实现留魂、留名、敌魂卡、神将收集、Boss 饶恕助战。不要把法宝做成卡牌。不要把封势写成格挡。第一批任务也暂时不要实现路线、商店、事件、完整法宝和 Boss。
```
