# T99：角色 / 敌人视觉方向与战斗占位替换计划

日期：2026-05-27
性质：视觉方向草案 + 程序化占位执行稿；本轮不生成 AI 图片，不引入正式素材

## 1. 目标

T99 的目标不是正式美术量产，而是在 T96 / T97 的新战斗 shell 上继续补足“第一眼识别”：

- 玩家能看出自己是三名原创执簿者之一，而不是一块状态文字。
- 敌方压案区能看出当前敌人类型，首战纸面鬼和 Boss 窃榜使有明显差异。
- 战斗画面继续服务残榜审案：敌人在案上受审，玩家在案下执簿，残榜中轴、三坛、法宝和手牌仍保持可读。

## 2. 边界

- 不生成 AI 图片。
- 不提交第三方图片、字体、音频、视频或素材包。
- 不把程序化占位当正式角色立绘或正式敌人立绘。
- 不新增玩法系统、角色、敌人、Boss、卡牌、法宝、章节或生产依赖。
- 不改变形、名、问名、正名、伏诛 / 归册、裁定、来势 / 异动、三坛或法宝规则。
- UI 可读性优先于视觉装饰：形、名格、来势 / 异动、手牌、香火和结束回合按钮不能被装饰压住。

若后续重新启用 AI 生图，所有图片必须先按 `docs/assets/ASSET_MANIFEST.md` 登记为“概念候选 / 不入正式构建”，授权、可商用和可修改均不得默认视为已确认。

## 3. 可玩角色视觉方向草案

### 3.1 衡简

- 身份锚点：旧榜执笔，残榜旧吏，稳健裁定入口。
- 轮廓：直立、收束、袖袍与卷轴同向，姿态像在案前按卷落笔。
- 材质：旧纸、乌木笔杆、暗金判印、少量青铜。
- 主色：旧金、墨绿、朱砂点印。
- 起始法宝呼应：打神鞭残节作为案边断鞭 / 直线符痕，不画成完整神兵。
- 程序化占位：玩家锚点使用竖卷轮廓、细长执笔剪影和一枚方正判印。
- 禁区：不要像姜子牙本人，不出现太公望式直接人物姓名或神将收集感。

### 3.2 照微

- 身份锚点：照影断名，低己形高信息，靠辨势和问名看破伪名。
- 轮廓：清瘦、锋利、镜面偏侧，视觉重心靠一枚照骨镜。
- 材质：冷玉、氧化铜镜、淡青镜光、细朱线。
- 主色：青绿、冷白、暗金。
- 起始法宝呼应：照骨镜作为圆形镜轮 / 斜切光，不做华丽仙侠光翼。
- 程序化占位：玩家锚点使用圆镜环、斜向光线和窄身剪影。
- 禁区：不要做成蓝紫仙侠法师，不用大面积神光遮挡 UI。

### 3.3 莲烬

- 身份锚点：莲骨破形，直攻爽感，高己形，火性与反噬边界。
- 轮廓：前倾、短促、燃尽的莲骨和赤绫火轮形成动势。
- 材质：焦莲骨、赤绫、炭黑边缘、朱红火印。
- 主色：朱红、炭黑、暗金。
- 起始法宝呼应：赤绫火轮作为环形火轮 / 绫带切线，不直接画哪吒。
- 程序化占位：玩家锚点使用火轮环、斜向绫带和更宽的冲刺剪影。
- 禁区：不要直接复刻哪吒形象，不出现儿童神将或抽卡立绘感。

## 4. 敌人与 Boss 视觉方向草案

### 4.1 纸面鬼

- 定位：第一战教学怪，服务形、破形、问名、正名、伏诛 / 归册教学。
- 轮廓：薄纸立起，边缘破裂，像错抄名页从案上浮起。
- 材质：旧纸纤维、墨渍、烧边、淡朱线。
- 主色：纸白、香灰、朱砂小痕。
- 程序化占位：敌人压案区使用撕纸剪影、两片纸页、细红封条和低压案影。
- 禁区：不做恐怖怪物大脸，不提高前三场教学压力。

### 4.2 窃榜使

- 定位：第一章 Boss，窃名、撕榜、遮名、塞污卷和终审压案的综合考核。
- 轮廓：高于普通敌人，像旧役吏与破榜阴影叠在一起；中轴有撕开的榜页。
- 材质：深红印泥、黑纸、残榜裂口、旧铜令牌。
- 主色：深红、黑褐、暗金、少量惨白纸边。
- 程序化占位：Boss 压案区使用高体量剪影、撕榜竖带、双重红印和更重的压案环。
- 禁区：不做可招募神将，不做饶恕助战暗示，不做多 Boss 池。

## 5. 本轮执行计划

1. 在 `docs/tasks/PROJECT_PLAN.md` 新增 T99，状态随本轮完成后收口。
2. 在 `BattleHud` 的 active battle shell 上追加 `t99-visual-placeholders` 标记。
3. 将底部 `battle-player-anchor` 扩成“角色视觉占位 + 文字状态”结构。
4. 按角色 ID 增加衡简 / 照微 / 莲烬三种程序化锚点。
5. 将敌方压案区增加更具体的身份 cue 和程序化层：重点覆盖纸面鬼与窃榜使。
6. 为 `capture:t99` 增加截图输出入口和 manifest 指标，确认程序化占位可见。
7. 同步素材台账：本轮仅新增程序化占位规格，未新增图片资产、AI 图或第三方素材。
8. 同步 PRD changelog、模板沉淀和 QA 记录。

## 6. 验收

- 首战截图能看出玩家锚点不是纯文字块，并能看出纸面鬼是纸页 / 残名方向。
- Boss 开场截图能看出窃榜使比普通敌人压迫感更强，具备窃榜 / 终审 / 撕榜视觉特征。
- 三名角色选择后进入战斗，玩家锚点至少通过形态、色彩和法宝符号表现差异。
- 战斗页仍可清楚读取己形、香火、手牌、敌人形、名格、来势 / 异动、后一动和操作按钮。
- 三档桌面视口无水平溢出；截图 manifest 能确认玩家视觉锚点和敌方身份 cue 可见。

## 7. 剩余风险

- 程序化占位只能解决第一眼识别和方向感，不能替代正式美术。
- 当前视觉仍与 CSS / Phaser 表现强绑定，后续若进入正式资产，需要重新拆分成 sprite / atlas / UI token。
- 角色视觉方向尚未经过用户审图定稿，不能进入商店页素材或宣传图生产。
