# PRD-pages：页面与交互需求

版本：v1.1
日期：2026-05-27
建议路径：`docs/prd/PRD-pages.md`

## 1. 页面总览

第一章公开 Demo 需要以下页面：

| 页面 | MVP 阶段 | 公开 Demo 阶段 |
|---|---|---|
| 标题页 | 简版可进入测试战斗 | 完整标题、继续游戏、设置、退出 |
| 战斗页 | 必须 | 完整核心 UI 与反馈 |
| 奖励页 | 简版 | 卡牌奖励、香火钱、稀有度、跳过；三角色版本开局为固定角色法宝，途中和 Boss 后法宝由三选一节点发放，候选排序可受角色和路线倾向影响 |
| 裁定页 | 朱批 MVP 后加入 | 登簿、朱批、削籍三选或事件化选择 |
| 路线页 | 阶段 B 简版 | 榜面节点地图、路线分支选择、路线倾向提示 |
| 商店页 | 阶段 C | seeded 货架、买卡、删卡、朱批、净卷 / 法宝维护、香封 |
| 休整页 | 阶段 C | 修整、删卡、朱批、法宝相关 |
| 事件页 | 阶段 C | 选项、代价、榜裂 / 劫数反馈 |
| 设置页 | 阶段 C | 音量、窗口、语言、动画开关 |
| 结算页 | 阶段 C | 伏诛 / 归册记录、裁定记录、败因或胜利 |

## 2. 标题页

MVP 只需要：

- 新开测试战斗。
- 开发者入口。
- 版本号显示。

公开 Demo 需要：

- 新游戏。
- 角色选择：第一章三角色版本在标题页选择衡简、照微或莲烬，并随角色带入固定起始法宝。
- 继续游戏。
- 设置。
- 退出。
- 标题视觉：一卷破裂的封神榜，朱砂线、烧痕、印章、模糊名籍。

## 3. 战斗页

战斗页核心布局：

- 顶部只保留极简资源条：第几战 / 遭遇名、墨、劫数、榜裂、设置。
- 主舞台左侧是执簿者形象、己形条和少量状态；衡简、照微、莲烬未来人物图均落在这里。
- 三坛属于玩家战斗节奏状态，贴在左侧执簿者信息下方，不占据敌我之间的动作通道。
- 主舞台中间保持为空，用作出牌飞行、破形、问名、正名、封势、异动等反馈通道，不放常驻大信息面板。
- 主舞台右侧是敌人形象、敌形条、来势 / 异动、名格和正名状态；敌方信息贴近敌人，不做横向大表格。
- 残榜不再作为常驻中央信息面板；常态只作为很淡的地面 / 桌面纹理，审案信息贴敌方，问名 / 正名 / 裁定时短暂展开。
- 临诏属于本场持续规则 / run 内战斗增益槽，放入底部快捷区，不常驻敌我之间。
- 法宝和香封以底部快捷或小挂件呈现，不混入普通手牌。
- 底部是唯一主要操作层：香火、手牌、临诏、法宝快捷、香封快捷、结束回合和轻量牌堆数量。
- 刚进入战斗页时，至少应能在首屏看到手牌卡面；不能只露出“手牌”标题。
- 战斗日志可展开。
- 术语 tooltip 可用。

必须显示：

- 执簿者己形 / 状态，贴近左侧执簿者形象。
- 当前香火，贴近底部手牌操作层。
- 手牌、抽牌堆、弃牌堆、消耗区数量，其中抽 / 弃 / 耗只需轻量显示。
- 敌人形，贴近右侧敌人形象。
- 敌人名格揭示状态，贴近敌人或敌人下方名札。
- 敌人本次行动类型：来势或异动。普通战默认明牌，遮势时只显示模糊信息；来势和异动必须是两个不同视觉标记。
- 当前来势值；遮势时不显示具体数值。来势标记必须支持封势反馈落在其上。
- 异动预警；遮势时不显示具体异动类型。异动必须从敌方侧发出，不和普通来势混淆。
- 辨势后的后一动预告槽：小、淡、条件展开，不抢本次行动主视觉。
- 已布置的坛位效果，以轻量标记或抽屉信息显示。
- 法宝状态：未认主 / 认主 / 过载 / 反噬，以快捷入口、角标或小挂件显示。

## 4. 术语展示

首次出现专有名词时，UI 必须使用“白话 + 术语”：

| 首次展示 | 后续可简写 |
|---|---|
| 敌人的形体（形） | 形 |
| 击破敌形（破形） | 破形 |
| 查明真名（问名） | 问名 |
| 真名已明（正名） | 正名 |
| 战后改榜（裁定） | 裁定 |
| 写入榜册（登簿） | 登簿 |
| 批改卡牌（朱批） | 朱批 |
| 抹去名籍（削籍） | 削籍 |

高级设置可以允许“术语简写模式”。

## 5. 战斗反馈

必须优先做的反馈：

- 破形：符火炸裂、纸面撕裂、轻微震屏。
- 问名：毛笔落纸，名格显现。
- 正名：朱砂印章落下，名破数值突出。
- 伏诛：煞气崩散。
- 归册：卷轴收束、青铜钟声。
- 登簿：榜册展开。
- 朱批：笔锋划过纸面。
- 削籍：刀刮纸面、榜册撕裂。
- 封势：符纸贴门，来势被压住。
- 异动：必须有明显预警，不要和普通来势混淆。
- 遮势：用弱化、半透明或虚线样式显示，不应误导为无行动。
- 后一动预告：仅在辨势后显示，位置低于本次行动，视觉权重低于本次行动。
- 法宝认主：金属清鸣。
- 法宝反噬：低频裂响。

MVP 可以用占位动效和占位音效，但事件必须清晰。

## 6. 奖励页

奖励页至少支持：

- 选择 1 张卡牌加入牌组。
- 跳过奖励。
- 显示伏诛 / 归册差异：两者卡牌奖励池一致、均为 3 选 1；归册的额外收益来自裁定。
- 根据 unlockState 过滤未解锁机制卡牌。

公开 Demo 增加：

- 钱 / 香火钱奖励。
- 显示卡牌稀有度：普通、精良、秘传；奖励结构为倾向位 + 基础位 + 惊喜位，但不在 UI 中暴露算法细节。
- 三角色版本开局由角色固定起始法宝承接；第一章途中和 Boss 后各保留 1 次法宝三选一。奖励页不把法宝混入普通卡牌奖励池。
- 三角色版本的卡牌奖励可以按当前角色构筑倾向排序，但不能因本场伏诛或归册改变奖励卡池和选项数量。
- 事件或精英后的裁定入口。

## 7. 裁定页

裁定页是本作最关键的成长页面。

只显示三类裁定：

- 写入榜册（登簿）。
- 批改卡牌（朱批）。
- 抹去名籍（削籍）。

不要出现留魂、留名、敌魂卡、Boss 助战。

裁定页必须显示：

- 本次归册对象。
- 已揭示的名。
- 可选裁定。
- 即时收益。
- 长期代价，例如榜裂。
- 当前榜裂、劫数、已登簿条目、已朱批卡。

T93 后，裁定页还需要显示：

- 精英 / Boss 登簿时的专属登簿名和规则文本，而不是只显示通用登簿。
- 削籍的具体判词变体：基础判词只常驻裂形符、抄裂得墨、预支香墨；重裂符只在裂榜针 / 终劫铃等裁定影响型法宝强化后显示，并按来源标明“法宝强化”。
- 预支香墨的延迟收益说明；若没有下一场战斗，该判词不应展示。
- 抄裂得墨除墨 +2 外，还需要说明下一次休整 / 事件里的朱批墨消耗 -1；休整页和事件页应在该减免存在时显示“朱批墨减免”状态，并只影响本来消耗墨的朱批入口。

## 8. 路线页

路线地图不是普通爬塔节点图，而是榜面上浮现的名字、印章、火烧洞、朱砂线。

节点类型：

- 普通战。
- 精英。
- 事件。
- 商店。
- 休整。
- Boss。

节点视觉像被写入或涂抹的名籍。

三主线版本的路线页必须显示多个可达节点，并用低成本标签说明路线倾向，例如稳行、归册、裂榜、补给、高压。路线提示只表达风险 / 收益 / 补给方向，不能暗示某个角色只能选择某条路线。

## 9. 商店页

商店公开 Demo 功能：

- 购买 seeded 货架卡牌，卡牌价格读取稀有度、费用和解锁阶段。
- 删除卡牌，首版按整局删牌次数递增价格 45 / 70 / 95。
- 购买临时朱批或裁定相关服务。
- 购买净卷服务、法宝维护服务或香封。
- 显示当前香火钱、货架类别、价格、是否已购、不可购买原因。
- 香封槽满时显示替换目标。
- 商品池必须跟随机制解锁。

当前第一章法宝节奏在三角色版本中调整为：开局随角色固定 1 件起始法宝，第一章过程中 1 次三选一，打完第一章 Boss 后 1 次三选一。途中和 Boss 后法宝 offer 可按角色与路线倾向排序，并在 UI 上显示构筑锚点标签。商店不直接出售法宝，避免第一章内法宝数量膨胀过快。旧单角色 fallback 可继续沿用开局三选一。

战斗页 T83 后还需要显示香封槽：香封是一次性战术资源，占槽、跨战斗携带、战斗内使用后消耗，不进入牌组。香封按钮必须显示目标需求、不可用原因和规则文本；使用、拒绝和消耗要进入 actionLog。

## 10. 休整页

休整点公开 Demo 功能：

- 恢复生命。
- 删除牌。
- 强化 / 朱批牌。
- 法宝相关操作，例如查看认主进度或清除一次反噬。

## 11. 设置页

至少支持：

- 主音量、音乐、音效。
- 窗口化、全屏、无边框窗口。
- 常用分辨率。
- 语言：中文，预留英文。
- 关闭动画 / 降低特效。
- 术语简写模式。

## 12. 结算页

结算页必须记录：

- 本局伏诛数量。
- 本局归册数量。
- 完成的裁定：登簿、朱批、削籍。
- 榜裂最终值。
- 劫数最终值。
- 法宝认主与反噬记录。
- Boss 是伏诛还是归册。
- 玩家失败原因或通关结果。

公开 Demo 结算要强化“我改写了一卷封神榜”的记忆点。
