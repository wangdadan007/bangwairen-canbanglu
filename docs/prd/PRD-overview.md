# PRD-overview：产品目标与版本范围

版本：v0.8
日期：2026-05-21
建议路径：`docs/prd/PRD-overview.md`

## 1. 产品定位

《榜外人：残榜录》是一款封神题材单机牌组构筑肉鸽游戏。

玩家扮演“封神榜外的执簿者”。封神大战之后，封神榜出现裂痕：若干神职空缺，若干姓名被涂抹，若干本该归位的名籍错乱。玩家进入残榜，在战斗中破敌之形、问敌之名，并在战后通过裁定改写残榜秩序。

核心表达不是“封神神将收集”，不是“商周战役复刻”，不是“普通打怪拿牌”，而是：

> 我在一场场战斗中查名、破形、正名、归册、裁定，逐步改写一卷破裂的封神榜。

### 1.1 题材来源与改编原则

本作的题材来源按三层处理：

- 人物、法宝、神职、典故主要从《封神演义》提取。它是最清楚、最稳定的叙事母本。
- 核心系统和世界观从“封神榜”这个物件提取，也就是名籍、榜册、登簿、削籍、错位、残榜，而不是照搬某一版“封神榜”电视剧的人物戏。
- 整体气质和市场标签使用“封神”这个大题材，可以吸收庙宇、香火、道教官署、民间信仰、神职秩序，但不要被某一版影视、动画或游戏形象绑死。

可玩角色可以借鉴高识别度封神人物的气质和机制原型，但第一章不直接让玩家扮演姜子牙、杨戬、哪吒、妲己、申公豹、闻仲等原典人物。首批可玩角色应是原创的“榜外执簿者”分支，避免滑向神将收集或商周阵营战役。

## 2. 产品支柱

### 2.1 封神榜编辑器感

玩家不是收集神将，而是在残榜中处理名籍、空缺、罪状、神职和榜裂。所有主要系统都必须服务“编辑封神榜”的体验。

### 2.2 破形与问名双轴战斗

战斗目标很清楚：把敌人的形打空。高级收益来自先问名、正名，再归册裁定。

### 2.3 裁定驱动的局内成长

归册后进入裁定，裁定只保留三类：登簿、朱批、削籍。它们分别对应本局规则、卡牌改造、高收益高代价。

### 2.4 东方仪式化 UI 与出牌节奏

美术、音频和 UI 必须服务残榜、朱砂、旧庙、青铜、木刻、香灰、暗金火光。战斗界面像桌案审判，卡牌像符诏与判词。

### 2.5 数据驱动和可测试

卡牌、敌人、遭遇、法宝、裁定、事件都应数据驱动。战斗逻辑必须和渲染逻辑分离，状态变化可追踪、可测试、可重放。

## 3. 目标玩家

- 喜欢单机牌组构筑肉鸽的玩家。
- 喜欢东方神话、封神、庙宇、残卷、审判感美术的玩家。
- 能接受中等学习成本，但需要清晰 UI 引导的玩家。
- 喜欢通过构筑把小牌改造成引擎的卡牌老玩家。

## 4. 明确不做

本项目在第一版 Demo 与后续正式版规划中都要主动避开：

- 不做传统封神神将收集。
- 不做商 / 周阵营战役复刻。
- 不做敌魂变卡。
- 不做 Boss 饶恕后助战。
- 不做“留魂”或“留名”系统。
- 不把卡面主语言写成“造成 X 伤害 / 获得 X 格挡”。
- 不让封势变成万能格挡。
- 不让三坛变成普通延迟伤害 / 延迟防御。
- 不做明显二次元封神手游、抽卡立绘、仙侠蓝紫渐变 UI。
- 不在宣传中使用“封神版杀戮尖塔”“类杀戮尖塔”“AI 生成游戏”等表达。

## 5. 第一版 Demo 产品目标

第一版 Demo 成功，不以内容数量为唯一标准，而以玩家是否记住以下体验为准：

- 敌人有形和名。
- 打空形就能赢。
- 问名、正名、归册能带来更高质量奖励。
- 裁定能改变本局成长路线。
- 朱批能让卡牌变成玩家自己的牌。
- 削籍有强诱惑，也会增加榜裂风险。
- 封势只能处理来势，异动需要专门应对。
- 三坛让出牌时机产生仪式感。
- 法宝认主让器物有成长，反噬让强度有代价。
- 三个原创可玩执簿者气质和机制方向不同，但都服务“改写残榜”这一主体验。
- 整体感觉像在编辑一卷破裂的封神榜。

## 6. 开发阶段

| 阶段 | 名称 | 目标 | 范围 |
|---|---|---|---|
| A | 核心战斗 MVP | 跑通抽牌、出牌、破形、问名、正名、伏诛/归册 | 1 角色、1 敌人、10-12 张基础卡、简单奖励 |
| B | 教学纵切 MVP | 连续 3-5 场战斗验证前半章节奏 | 3-5 敌人、15-20 张卡、朱批、人坛、基础 UI |
| C | 第一章公开 Demo | 对外试玩级完整第一章 | 3 个原创可玩执簿者、42-48 张卡、12 普通敌人、3 精英、1 Boss、10-12 法宝、8-10 事件 |

本阶段表描述开发路线，不表示当前实时进度。当前已完成任务、推荐下一步和阶段判断必须读取 `docs/tasks/PROJECT_PLAN.md`。

## 7. 正式版章节规划

内部规划采用 4 个常规章 + 1 个终审短章，共 5 章。对外宣传阶段暂不承诺完整章节数。

| 章 | 暂定名称 | 功能定位 |
|---|---|---|
| 第一章 | 残榜初裂 | 核心机制教学与 Demo 章节 |
| 第二章 | 诸司错位 | 神职错乱，登簿和墨资源深化 |
| 第三章 | 香火逆流 | 人间庙宇、香火、愿鬼、异动压力 |
| 第四章 | 万名相噬 | 榜裂、削籍、篡名、法宝反噬加压 |
| 第五章 | 封榜终审 | 短终局章，按本局登簿、朱批、削籍、榜裂生成最终审判 |

第五章可以是 3 到 5 个节点 + 最终 Boss，不必做成完整大地图。
