# PRD-codex：Codex 使用规则与首批任务

版本：v0.5  
日期：2026-05-16  
建议路径：`docs/prd/PRD-codex.md`

## 1. Codex 任务原则

Codex 主要负责开发执行。每次任务必须小步推进，不要一次性重写全项目。

Codex 在执行前必须阅读：

- `AGENTS.md`
- `docs/prd/INDEX.md`
- `docs/prd/PRD-overview.md`
- 本任务相关 PRD 文件
- 最近的 `docs/prd/PRD-changelog.md`

如果任务涉及战斗机制，必须读 `PRD-gameplay.md`。  
如果任务涉及内容规模，必须读 `PRD-content-scope.md`。  
如果任务涉及 UI，必须读 `PRD-pages.md`。  
如果任务涉及验收，必须读 `PRD-checklist.md`。

## 2. Codex 输出格式

每次任务完成后，Codex 必须输出：

- 本次目标。
- 修改文件列表。
- 关键实现说明。
- 已完成内容。
- 未完成风险。
- 如何运行和测试。
- 下一步建议。

## 3. 开发顺序

推荐顺序：

1. 技术骨架：TypeScript / Vite / React / Phaser，完成项目初始化、数据加载、基础场景切换。
2. 基础战斗循环：抽牌、费用、出牌、弃牌、回合结束、敌人行动。
3. 形与破形：敌人形、破形结算、形归零、胜负判断。
4. 来势、封势、异动：封势只能处理来势，异动必须用专门逻辑处理。
5. 名、问名、正名、名破：实现名格、揭示、正名触发、名破削形。
6. 伏诛 / 归册结算：同样形归零，但根据是否正名进入不同奖励质量。
7. 第一批 12 到 15 张卡 + 前 3 场教学战。
8. 裁定 MVP：先做朱批，再做登簿，最后做削籍 / 榜裂。
9. 路线地图、奖励、商店、休整、事件。
10. 墨、劫数、榜裂。
11. 三坛：先人坛，再地坛，再天坛。
12. 法宝基础效果、认主、反噬。
13. 精英与 Boss。
14. 存档、继续游戏、设置、结算页。
15. 美术 / 音效占位和动效手感贯穿开发，最后阶段替换和强化。
16. 桌面打包与 Steam Demo 准备。

## 4. 首批任务列表

### DEV-001：建立项目技术骨架与核心战斗状态模型

优先级：P0  
所属模块：技术骨架 / 战斗核心

目标：建立 TypeScript + Vite + React + Phaser 项目骨架，实现与渲染解耦的核心战斗状态模型。

必须实现：

- 初始化前端项目，使用 TypeScript。
- 接入 Phaser 作为战斗场景渲染层。
- React 负责菜单、调试面板或外层 UI。
- 建立 `src/core` 纯逻辑层，不依赖 Phaser / React。
- 建立基础数据结构：CardDefinition、EnemyDefinition、BattleState、PlayerState、EnemyState、ActionLog。
- 实现开局抽牌、获得香火、出牌、弃牌、结束回合、敌人行动、进入下一回合。
- 实现“形”和“破形”：敌人形归零后战斗胜利。
- 准备 5 到 8 张占位卡和 1 个占位敌人“纸面鬼”。
- 所有核心状态变化写入 actionLog。
- 添加基础测试，至少覆盖破形、费用不足、抽牌弃牌循环、敌人形归零胜利。

不做：完整美术、路线、商店、事件、法宝、裁定、Boss。

验收：浏览器可进入测试战斗；玩家可打出基础破形牌；敌人形正确减少；形归零显示胜利；核心逻辑与 React / Phaser 解耦；测试通过或说明失败原因。

### DEV-002：实现问名、正名、名破、伏诛 / 归册

优先级：P0  
所属模块：战斗核心

目标：实现名格、问名、无名 fallback、正名、名破和两类结算。

必须实现：

- 敌人配置 nameSlots。
- 问名牌揭示名格。
- 无名目标问名时改为辨势或弱破形。
- 全部名格揭示后进入正名。
- 正名触发名破。
- 未正名形归零显示伏诛。
- 正名后形归零显示归册。
- actionLog 记录全部关键事件。

### DEV-003：实现来势、封势、异动边界

优先级：P0  
所属模块：敌方行动

目标：实现敌人行动分层，并在代码层保证封势不能处理异动。

必须实现：

- EnemyIntent 支持 incomingForce 与 abnormalMove。
- 封势只降低 incomingForce。
- 至少一个异动：偷香或塞污卷。
- 至少一个应对异动的卡牌。
- UI 或调试面板明确显示来势与异动。

### DEV-004：实现前三场教学战纵切

优先级：P0  
所属模块：教学 / 内容

目标：连续跑通前 3 场固定教学战。

必须实现：

- 第 1 场：纸面鬼，教学形、破形、问名、正名、伏诛、归册。
- 第 2 场：偷香鼠，教学来势、异动、封势、断供。
- 第 3 场：铜铃夜巡，教学朱批入口或朱批雏形。
- 奖励池跟随机制解锁。
- 未解锁机制不出现在奖励中。

### DEV-005：实现裁定 MVP

优先级：P0  
所属模块：局内成长

目标：归册后进入裁定，先实现朱批，再实现登簿，最后实现削籍与榜裂。

必须实现：

- 裁定页入口。
- 朱批：对已有卡牌添加永久词条。
- 登簿：添加本局永久规则。
- 削籍：强收益 + 榜裂增加。
- 不出现留魂、留名、敌魂卡、Boss 助战。

### DEV-006：实现法宝基础系统

优先级：P1  
所属模块：法宝

目标：实现法宝作为牌组外器物，不进牌组，具备未认主和认主状态。

必须实现：

- ArtifactDefinition。
- artifactState。
- 法宝栏显示。
- 至少 4 件法宝：打神鞭残节、照骨镜、朱砂斗、裂榜针。
- 认主条件与认主效果。
- 过载 / 反噬字段预留。

## 5. DEV-001 可直接复制给 Codex 的 prompt

```text
请为《榜外人：残榜录》建立第一阶段开发骨架。技术方向为 TypeScript + Vite + React + Phaser。当前目标不是完整游戏，而是核心战斗 MVP。

请先阅读 AGENTS.md、docs/prd/INDEX.md、docs/prd/PRD-overview.md、docs/prd/PRD-architecture.md、docs/prd/PRD-gameplay.md、docs/prd/PRD-checklist.md 和 docs/prd/PRD-codex.md。

请实现一个数据驱动、逻辑和渲染分离的基础战斗系统：src/core 只包含纯战斗逻辑，不依赖 Phaser 或 React；src/game 使用 Phaser 显示测试战斗场景；src/ui 可用 React 显示菜单或调试面板。

战斗核心概念：玩家每回合获得香火，抽牌，打出卡牌。敌人有“形”，卡牌使用“破形 X”减少敌人的形。敌人形归零是唯一基础胜利条件。请先实现破形、费用、抽牌、弃牌、回合推进、敌人行动和胜利判断。问名、正名、伏诛、归册、裁定本次只预留字段和接口，不完整实现。

请建立基础数据文件 cards.json、enemies.json、encounters.json、tutorial_unlocks.json，并提供 5 到 8 张占位卡和 1 个敌人“纸面鬼”。所有状态变化应记录到 actionLog，方便测试和回放。请添加基础测试，覆盖出牌费用、破形结算、抽弃牌循环、敌人形归零胜利。

验收要求：npm install 后可以运行开发服务器；浏览器中可以进入测试战斗；玩家可以打出卡牌破形；敌人形归零后显示胜利；核心逻辑与 Phaser / React 解耦；测试通过或明确说明失败原因。

完成后请输出：本次目标、修改文件列表、关键实现说明、已完成内容、未完成风险、如何运行和测试、下一步建议。
```
